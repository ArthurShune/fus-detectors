\begin{abstract}
We present a fixed-parameter spatiotemporal adaptive processing (STAP) operating profile for functional ultrasound (fUS), evaluated primarily in simulation, together with deployment-oriented telemetry and a conservative runtime gating layer. The STAP profile fixes tile geometry, temporal aperture, Doppler bands, and covariance conditioning, and optionally supports conditional execution that skips STAP on tiles excluded by a flow-proxy mask. We implement the gating layer as an auditable, non-increasing score veto governed by a three-state runtime contract: C0 (telemetry-only; KA off), C1 (safety-eligible; bounded shrink on high-risk, low-confidence tiles), and C2 (research-only; requires additional label-free actionability evidence). The contract uses label-free telemetry (band energies/ratios, flow coverage, guard fractions) and emits explicit reason codes so that ``KA does nothing'' is an expected success mode when Pf/Pa structure is absent or non-actionable. For labeled Brain-* acceptance bundles we report low-FPR ROC using the exported right-tail PD score map $S_{\mathrm{PD}}$ and state operating points as $S_{\mathrm{PD}}\ge\tau$ calibrated by $\mathbb{P}\{S_{\mathrm{PD}}\ge\tau\mid \mathrm{bg}\}=\alpha$.

In k-Wave brain fUS simulations (Brain-OpenSkull, Brain-AliasContract, Brain-SkullOR), the fixed STAP profile improves low-FPR detection over MC--SVD/\allowbreak RPCA/\allowbreak HOSVD under this scoring protocol. Conditional execution reduces latency by skipping tiles and falling back to the baseline there; because this changes the decision rule and uses a flow-proxy mask, it is treated as a compute heuristic requiring separate validation for information leakage. On two real datasets without task labels (PD-only whole-brain mouse fUS and microbubble IQ clips), we report end-to-end determinism, telemetry behavior, contract activation rates, and label-free stability/hygiene proxies rather than detection efficacy.
\end{abstract}

\section{Introduction}

We study slow-time detection on fUS tiles under fixed-parameter, dataset-class operating profiles and an explicit runtime contract governing an optional knowledge-aided (KA) layer. Quantitative detection results in this document are restricted to labeled simulation regimes; real-data experiments are reported as telemetry/instrumentation (contract behavior and label-free proxies) and are not evidence of clinical efficacy.
By \emph{fixed profile} we mean that once an operating profile is selected for a dataset class (Brain-* simulations, Shin kHz IQ, or PD-only hemodynamics), its tile geometry, bands, conditioning targets, and contract thresholds are reused unchanged across seeds/clips/planes; only deterministic preprocessing (e.g.\ PD-derived proxy masks) and a one-time label-free baseline selection for Shin are permitted. Baseline clutter suppression (SVD/\allowbreak MC--SVD/\allowbreak RPCA/\allowbreak HOSVD) yields a whitened slow-time vector $z = \wh R^{-1/2} x$; a temporal STAP core scores tiles in fixed flow/alias bands $(\Pf,\Pa)$ using BR/GLRT-type ratios and produces PD-derived maps for scoring and proxying. We distinguish conventional PD \emph{brightness} $\mathrm{PD}^{\uparrow}$ (higher indicates more power; used only for deterministic preprocessing) from the exported right-tail PD score $S_{\mathrm{PD}}$ used for low-FPR ROC reporting (Notation). Knowledge-aided processing is treated as an optional, layered mechanism. For completeness we describe a band-aware covariance reshape $z' = R_\beta^{-1/2}x = Qz$ and the classical feasibility conditions (R--1/R--2/R--3) under which such transforms could yield low-FPR ROC uplift. R--1/R--2/R--3 are design-time feasibility conditions used to gate uplift claims on labeled benchmarks; the C0/C1/C2 contract is a runtime safety/eligibility mechanism that can disable KA even when feasibility might hold. For deployment we implement a score-space KA layer (Section~\ref{sec:ka-contract-v2}) that is shrink-only: it can only reduce suspicious activations and preserves high-confidence flow tiles. A three-state runtime contract (C0/C1/C2) uses label-free telemetry to decide when KA is off (success mode) versus when mild, auditable vetoes are allowed.

\begin{itemize}[nosep]

  \item \textbf{Allowed one-time calibration (label-free).} For Shin we sweep the MC--SVD energy-fraction baseline cutoff on a small calibration subset and freeze a single value based on telemetry integrity/stability (not ROC).   \item \textbf{Descriptive thresholds (not deployable).} Some PD-only Mac{\'e} summaries use atlas labels to choose per-plane PD-z thresholds for descriptive visualization of gating effects; these are not used as deployable operating points and are not evidence of efficacy.
  \item \textbf{Not tuned per seed/clip/plane.} We do not retune STAP/KA hyperparameters per seed/clip/slice to maximize headline ROC; any subset selection performed on the same dataset is stated explicitly.
\end{itemize}

\subsection{Evaluation protocol and uncertainty reporting}
\label{sec:eval}

We report operating points at fixed low false-positive rates (FPR) using the right-tail PD score map $S_{\mathrm{PD}}$ (Notation). For a given method, regime, and target $\alpha$, we choose a threshold $\tau$ so that $\mathbb{P}\{S_{\mathrm{PD}}\ge\tau\mid \mathrm{bg}\}=\alpha$ on that method's negative set, then report $\mathbb{P}\{S_{\mathrm{PD}}\ge\tau\mid \mathrm{flow}\}$ on its positive set. Unless otherwise stated, the positive/negative sets are the fixed masks exported in the corresponding acceptance bundle. When we report uncertainty across seeds or windows we use bootstrap resampling at the natural unit of variation (seed for multi-seed pilots; window for non-overlapping-window ablations), and use paired bootstraps when comparing two methods on the same seed/window.

\paragraph{Contributions.}
\begin{itemize}[nosep]
  \item A fixed-parameter STAP operating profile (tile geometry, temporal aperture, Doppler bands, conditioning targets) evaluated on labeled Brain-* k-Wave simulations, with low-FPR ROC reporting under a stated PD-mode scoring protocol.
  \item A conservative runtime gating contract v2 (C0/C1/C2) implemented as an auditable, non-increasing score veto with protected-set invariants and explicit reason codes; ``KA off'' is an expected outcome out of regime.
  \item A conditional execution option that skips STAP on tiles excluded by a flow-proxy mask and falls back to the baseline there, reported as a compute heuristic requiring separate validation for information leakage.
  \item Real-data instrumentation on PD-only whole-brain mouse fUS and unlabeled kHz IQ clips: telemetry behavior, contract activation rates, and label-free stability/hygiene proxies (not detection efficacy).
  \item Bridge and stress-test datasets showing contract-negative behavior when Pf/Pa structure is not realized, motivating evaluation on labeled raw-IQ acquisitions.
\end{itemize}

\subsection*{System overview}

Algorithmically, the detector we study is more than a clutter filter: it is a complete slow-time operating regime for functional ultrasound:
\begin{itemize}[nosep]
  \item A fixed-parameter STAP profile for brain-like fUS regimes, in which a motion-compensated SVD clutter filter provides a baseline PD-derived map (a conventional brightness proxy $\mathrm{PD}^{\uparrow}$ for deterministic mask/proxy construction when needed) and an adaptive temporal STAP core operates on small slow-time tiles. For ROC reporting we use the exported right-tail PD score $S_{\mathrm{PD}}$ stored as \texttt{score\_pd\_*.npy} (Notation); bundles may also include legacy \texttt{pd\_*.npy} maps for compatibility.
  \item A \emph{banded telemetry layer} that exposes per-tile flow/alias/guard energies and peak-band occupancy using fixed, disjoint projectors $(\Po,\Pf,\Pg,\Pa)$; these telemetry channels are used both for scientific interpretation (``which band dominates where?'') and for contract decisions.
  \item A \emph{contract-driven KA layer} with two roles: (i) a band-aware covariance reshape described for completeness and feasibility analysis (operator-level structure, R--1/R--2/R--3), and (ii) a score-space KA contract v2 implemented as a shrink-only, auditable veto. The v2 contract emits a state (C0/C1/C2), a reason code, and a risk mode (alias vs guard) so that ``KA does nothing'' is a designed outcome when telemetry is flat or non-actionable.
  \item An optional \emph{conditional execution} mode that separates baseline clutter suppression, temporal STAP, and knowledge-aided transforms. In this mode, STAP is applied only on tiles that intersect a flow-proxy mask (derived from a baseline PD-brightness proxy $\mathrm{PD}^{\uparrow}$ when available, or from simulator geometry), while tiles with no flow support retain the baseline output (no STAP). We treat this as a compute heuristic that changes the decision rule and therefore requires separate leakage validation.
  \item A hierarchy of \emph{simulated and real regimes} that exercise both detection and contract logic: Brain-* k-Wave regimes for ROC/latency and alias stress tests; PD-only Mac{\'e} whole-brain fUS for hemodynamic telemetry and sentinel behavior; real kHz-rate IQ (Shin RatBrain Fig3) as an end-to-end viability and safety anchor with label-free hygiene endpoints; MaceBridge Doppler simulations and public datasets as contract-negative stress tests that highlight data limitations.
\end{itemize}

The remainder of the document summarizes the fixed-profile pipeline and evaluation protocol, reports simulation detection results and real-data instrumentation outcomes under the stated claims discipline, and provides modality-agnostic contract theory and implementation details in appendices.

Throughout, $H_1$ denotes ``flow'' tiles and $H_0$ background/no--flow tiles.

\subsection*{Notation}

\begin{itemize}[nosep]
  \item Slow--time length $T$. 
  \item Disjoint band projectors $(\Po,\Pf,\Pg,\Pa)$ (other/DC/clutter, flow, guard, alias) with $\Po+\Pf+\Pg+\Pa=I$.
  \item Baseline covariance estimate $\wh R$; KA blended covariance $R_\beta$.
  \item Baseline whitened tile $z = \wh R^{-1/2}x$. KA whitened tile $z' = R_\beta^{-1/2} x = Qz$.
  \item PD scoring convention (repository). Acceptance bundles store a conventional PD brightness proxy
  \[
    \mathrm{PD}^{\uparrow}(y,x)=\frac{1}{T}\sum_{t=1}^{T}\left|X_{\mathrm{base}}(t,y,x)\right|^2,
  \]
  used only for deterministic preprocessing (e.g.\ flow-proxy masks). For low-FPR ROC reporting we use the exported right-tail PD score map
  \[
    S_{\mathrm{PD}}(y,x)\equiv\texttt{score\_pd}(y,x),
  \]
  and state operating points as $S_{\mathrm{PD}}\ge\tau$ with $\mathbb{P}\{S_{\mathrm{PD}}\ge\tau\mid \mathrm{bg}\}=\alpha$ (with $\tau$ chosen per method on that method's negative set). Newer bundles export \texttt{score\_pd\_*.npy} explicitly; when only legacy \texttt{pd\_*.npy} maps are present, we define $S_{\mathrm{PD}}=-\texttt{pd}$ (equivalently, lower-tail thresholding on \texttt{pd}). This sign convention is verified by \path{scripts/pd_mode_sanity.py} and unit tests. In \texttt{score\_mode=pd}, \texttt{pd\_stap.npy} is obtained by modulating \texttt{pd\_base.npy} by a per-tile flow-band fraction and clamping background/conditional-execution skips back to baseline (see \path{sim/kwave/common.py:_stap_pd_tile_lcmv}).
  \item Conjugate-transpose $(\cdot)\Hh$ (equal to $(\cdot)^\top$ for real-valued PD-only instantiations).
  \item For a Hermitian $A$, band--restricted extreme eigenvalues:
  \[
    \lambda_{\min}(A|_S) := \min_{\substack{v\in\mathrm{ran}(S)\\ \Norm{v}=1}} v\Hh A v,\quad
    \lambda_{\max}(A|_S) := \max_{\substack{v\in\mathrm{ran}(S)\\ \Norm{v}=1}} v\Hh A v.
  \]
  \item Commutator: $[Q,\Pf] = Q\Pf - \Pf Q$, and similarly for $\Pa$.
\end{itemize}

\section{Runtime contract v2 (score-space veto)}
\label{sec:ka-contract-v2}

We implement knowledge-aided (KA) processing as an optional, conservative score-space veto governed by a three-state runtime contract (C0/C1/C2) driven by label-free telemetry. C0 disables KA and records telemetry only; C1 enables a bounded, shrink-only veto on a localized set of high-risk, low-confidence tiles while protecting high-confidence flow tiles; C2 is a research-only state that additionally requires label-free \emph{actionability} evidence before applying stronger vetoing.

Operationally, the contract is defined on per-tile telemetry (band energies/ratios, guard fractions, and flow-proxy coverage) and acts on a right-tail score $S_{\mathrm{base}}$ (in PD-mode, $S_{\mathrm{base}}=S_{\mathrm{PD}}=\texttt{score\_pd}$). When enabled, it produces a non-increasing veto on this score. In PD-mode bundles we export a per-pixel scale map $\gamma(i)\ge 1$ (stored as \texttt{ka\_scale\_map.npy}) and apply it to the stored PD map as $\mathrm{PD}_{\mathrm{post}}(i)=\gamma(i)\,\mathrm{PD}_{\mathrm{pre}}(i)$ on gated regions (and $\gamma=1$ on protected regions); since $S_{\mathrm{PD}}=-\texttt{pd}$, this is a pointwise non-increasing transformation of $S_{\mathrm{PD}}$. For any fixed decision threshold $\tau$ chosen upstream, this mapping is a pure veto (it can only remove pixels from $\{S_{\mathrm{PD}}\ge\tau\}$); when reporting low-FPR ROC points where $\tau$ is reselected post-KA to satisfy $\mathbb{P}\{S_{\mathrm{PD}}\ge\tau\mid \mathrm{bg}\}=\alpha$, changes in TPR reflect background-tail suppression despite scores never increasing. The contract emits explicit reason codes so that ``KA off'' is an expected success mode under flat or non-actionable telemetry.

\SuppOrApp{app:ka-contract-theory} provides the modality-agnostic sufficient conditions, score-level analysis, and additional sentinels used to interpret when KA should be enabled or disabled.

\subsection{Mac{\'e} whole-brain fUS: hemodynamic telemetry and hold-out gating check}
We treat the Mac{\'e}/Urban whole-brain dataset as a PD-only, open-skull analogue of Brain-OpenSkull and run a tile-level hemodynamic telemetry analysis with fixed bands $\Pf=[0,0.5]$~Hz, guard $[0.5,1.0]$~Hz, and $\Pa=[1.0,3.0]$~Hz on $8\times 8$ tiles (stride 3). Offline we also compute atlas-defined tile labels (VIS/SC/LGd as H$_1$, CP/CA/DG as H$_0$) for descriptive summaries; these atlas labels are not used as contract inputs. Figure~\ref{fig:mace_phase2_summary} summarizes a systematic sweep in ``Pf occupancy vs alias separation'' space.
Here PD-z denotes a per-tile change-from-baseline z-score on the PD time series (distinct from the Doppler/IQ acceptance-bundle PD score maps $S_{\mathrm{PD}}$).

\paragraph{Hold-out discipline (avoid plane selection bias and label-tuned thresholds).}
To address the selection-bias critique and to avoid per-plane PD-z thresholds chosen using atlas labels (e.g.\ forcing $\mathrm{TPR}\approx 0.5$), we add a scan-level hold-out evaluation implemented in \path{scripts/mace_alias_gate_holdout.py}. We (i) define label-free proxy sets from a mean-PD vascularity mask and set a per-plane PD-z operating point by a fixed background-proxy tail rule $\mathbb{P}\{\mathrm{PD}\text{-}z\ge\tau\mid \mathcal{T}_{\mathrm{bg}}\}=\alpha_{\mathrm{pd}}$ (no atlas labels), (ii) define the alias veto threshold as a fixed background-proxy quantile of $m_{\mathrm{alias}}=\log((E_{\mathrm{Pa}}+\epsilon)/(E_{\mathrm{Pf}}+\epsilon))$, and (iii) select ``contract-positive'' planes in held-out scans using a label-free telemetry statistic (Pf-peak fraction on flow-proxy tiles), with the selection threshold fixed from the training scans. Table~\ref{tab:mace_holdout_alias_gate} summarizes the resulting held-out counts on \texttt{scan4}--\texttt{scan5} (training: \texttt{scan1/2} and \texttt{scan3/6}; note \texttt{scan2} duplicates \texttt{scan1} and \texttt{scan6} duplicates \texttt{scan3} in this loader).

On held-out scans (\texttt{scan4}--\texttt{scan5}), this fixed, shrink-only alias veto reduces offline H$_0$ detections but also removes offline H$_1$ detections at the chosen label-free operating point. At $\alpha_{\mathrm{pd}}=0.05$ on the background proxy, aggregate (H$_1$/H$_0$ detections) change from $300/180$ to $239/128$ (detections retained $\approx 0.80$). On the telemetry-selected subset of held-out planes (PfPeak$_{\mathrm{flow}}\ge\tau_{\mathrm{sel}}$), detections change from $47/43$ to $43/38$ (detections retained $\approx 0.91$). We interpret this as a conservative ``safety veto'' behavior rather than an efficacy claim: Mac{\'e} is PD-only and provides no Doppler-side labels, so its primary role here is to validate the telemetry/proxy machinery and the no-peeking threshold discipline.

\subsection{Shin RatBrain Fig3 (LOCA-ULM): real kHz IQ viability and contract dashboard}
\label{sec:shin-ratbrain}

To demonstrate that the frozen STAP + telemetry + contract stack is operational on real Doppler-like IQ, we use the LOCA-ULM RatBrain Fig3 dataset released with Shin et al.\ as beamformed, post-compounded complex IQ sequences \cite{ShinLOCAULM2024,ShinZenodo10711806}. This is a microbubble ULM IQ dataset; we use it solely as a kHz-IQ ingestion/telemetry/contract robustness anchor, not as a physiological fUS benchmark. The Fig3 archive contains 80 clips (\texttt{IQData001.dat}--\texttt{IQData080.dat}) and a \texttt{SizeInfo.dat} file describing the tensor shape. Each clip stores $T=250$ slow-time frames on a $(238\times 130)$ spatial grid as little-endian float64 arrays containing I then Q in column-major order (matching the provided \texttt{LoadIQ.m}); the effective slow-time sampling is $1$~kHz (post-compounding), with center frequency $f_0\approx 15.6$~MHz \cite{ShinLOCAULM2024}. We treat this dataset as a \emph{real-IQ viability and safety anchor} rather than a labeled detection benchmark: there are no $H_1/H_0$ labels, so we do not report ROC uplift; instead we report contract stability (tilewise telemetry/coverage), shrink-only invariants, and label-free endpoints defined on pixel-level proxy masks (flow/background).

\paragraph{Bundle writer and pipeline.}
We mirror the authors' loader in \path{pipeline/realdata/shin_ratbrain.py} and convert IQ clips into standard acceptance bundles using \path{sim/kwave/icube_bundle.py:write_acceptance_bundle_from_icube}. For a chosen clip and time window we:
\begin{itemize}[nosep]
  \item Apply an SVD clutter filter on the Casorati matrix $X\in\mathbb{C}^{T\times HW}$ to form a baseline PD-derived map $pd_{\mathrm{base}}$ and a residual IQ cube (the input to STAP).
  \item Construct pixel-level proxy masks from $pd_{\mathrm{base}}$ (saved as \texttt{pd\_base.npy}; used here as a conventional PD-brightness proxy $\mathrm{PD}^{\uparrow}$ distinct from the ROC score maps): a flow/vessel proxy (top quantile; minimum pixel count) and a background proxy (complement of the flow mask in a central crop).
  \item Run the temporal STAP PD core on the residual IQ cube with fixed tile geometry ($8\times 8$ tiles, stride 3) and a longer slow-time aperture $L_t=64$ suited to $1$~kHz slow time (bin spacing $15.6$~Hz), producing a pre-KA PD-mode map (saved as \texttt{pd\_stap\_pre\_ka.npy} and \texttt{score\_pd\_stap\_pre\_ka.npy}).
  \item Apply the KA contract v2 on tilewise telemetry and the PD scoring convention (operate on the right tail of the PD score $S_{\mathrm{PD}}$) to produce a contract state (C0/C1/C2), reason code, and (if enabled) a non-increasing veto map, producing $pd_{\mathrm{stap}}$ (saved as \texttt{pd\_stap.npy} and \texttt{score\_pd\_stap.npy}) and the KA maps (\texttt{ka\_gate\_map.npy}, \texttt{ka\_scale\_map.npy}).
  \item Record tilewise telemetry maps (peak frequency, guard fraction, alias/guard risk metrics) and the KA maps (\texttt{ka\_gate\_map.npy}, \texttt{ka\_scale\_map.npy}) in the same bundle format used throughout the repository.
\end{itemize}
The proxy masks are used only for label-free proxying/protection; contract decisions and hygiene/robustness proxies follow the PD scoring convention used elsewhere in this repository: operating points are expressed as a right-tail threshold on the exported score map $S_{\mathrm{PD}}$, i.e.\ $S_{\mathrm{PD}}\ge\tau$ at fixed background tail rate. (When explicit \texttt{score\_pd\_*.npy} maps are not present, we use $S_{\mathrm{PD}}=-\texttt{pd}$ by construction.)

\emph{Scale map convention.} In our acceptance bundles, \texttt{ka\_scale\_map.npy} stores a scale factor $\gamma(i)\ge 1$ and is applied to legacy \texttt{pd} maps as
\[
\mathrm{PD}_{\mathrm{post}}(i)=\gamma(i)\,\mathrm{PD}_{\mathrm{pre}}(i),\qquad \gamma(i)\ge 1,
\]
which makes $S_{\mathrm{PD}}=-\texttt{pd}$ non-increasing on gated regions. For any fixed threshold $\tau$ this is a pure veto; when thresholds are recalibrated post-KA to match a target background tail rate, any TPR change reflects background-tail suppression rather than score inflation.

\paragraph{Frozen Shin operating profiles and baseline calibration.}
To avoid retuning the conservative brain Doppler profile (\texttt{--stap-profile clinical} in code) to Shin until outcomes appear favorable, we treat Shin as its own frozen operating regime and explicitly calibrate only the baseline clutter filter once. We sweep the MC--SVD energy-fraction cutoff (smallest number of singular modes removed to exceed the target energy fraction) over a calibration subset (10 representative clips; window $0{:}128$) using \path{scripts/shin_ratbrain_profile_sweep.py} and select a single frozen baseline setting based on telemetry integrity and stability, not on contract-state counts. Table~\ref{tab:shin_baseline_sweep} reports C1/C2 counts for transparency, but selection is based on the label-free integrity/actionability metrics ($\Delta_{bf}$, $\Delta_{\mathrm{tail}}$) and stability/hygiene proxies. Overly aggressive baseline removal increases Pf peak occupancy but can invert actionability telemetry (background-vs-flow separation becomes negative) and suppress C2 eligibility. We therefore freeze MC--SVD with energy fraction $\approx 0.97$ as a balanced setting for the Shin-U profile.

\paragraph{Profile variants (Shin-U vs Shin-S/L).}
In addition to Shin-U, we evaluated two alternative frozen band profiles (Shin-S and Shin-L) designed to push the ``flow'' band Pf toward lower frequencies and increase Pf peak occupancy (an interpretability-forward choice for microvascular-like content under 1~kHz slow time). Across the same 10-clip calibration subset, these variants typically increase Pf-peak(flow) but render the v2 actionability check $\Delta_{\mathrm{tail}}$ negative under our flow/background proxy sets, so the contract remains predominantly in C1 (safety-only) rather than C2 (uplift-eligible). We therefore treat Shin-S/L as conservative ``deployment-mode'' profiles (C1-only by design) and use Shin-U as the primary ``real IQ viability + safety'' anchor reported here.

\paragraph{Contract behavior, stability, and hygiene endpoints.}
Under the frozen Shin-U profile (MC--SVD energy fraction $\approx 0.97$, STAP $L_t=64$, fixed Pf/Pa telemetry bands, KA v2 enabled in \texttt{auto} mode), we run the pipeline on 10 pre-selected clips (\texttt{IQData001}--\texttt{IQData005}, \texttt{IQData010}, \texttt{IQData020}, \texttt{IQData040}, \texttt{IQData060}, \texttt{IQData080}) and evaluate stability across three overlapping time windows ($0{:}128$, $64{:}192$, $122{:}250$) using \path{scripts/shin_ratbrain_hygiene_metrics.py}. Because there are no labels, we report only label-free endpoints:
\begin{itemize}[nosep]
  \item \textbf{Contract stability.} C1 dominates in all windows (8--10 of 10 clips), while C2 is rare (0--2 clips) due to the Pf-realization guardrail for C2 based on flow-proxy tiles (tiles with high flow-mask coverage); all decisions emit auditable reason codes.
  \item \textbf{Shrink-only and protected invariance.} When KA is active it applies a mild, localized shrink (median $p_{\mathrm{shrink}}\approx 0.09$; median scaled pixel fraction $\approx 0.20$) and preserves the protected set exactly (maximum absolute PD change on protected pixels is $0$ in all evaluated bundles).
  \item \textbf{Background-tail hygiene.} A label-free background-tail metric shows consistent tail compression on early windows: for each bundle we choose a threshold $\tau$ so that the background proxy satisfies $\mathbb{P}\{S_{\mathrm{PD}}\ge\tau\mid \text{bg proxy}\}=10^{-3}$ on the pre-KA map, then count clusters/area of pixels satisfying $S_{\mathrm{PD}}\ge\tau$ before vs.\ after KA. This is interpreted as ``map hygiene'' rather than ROC uplift.
\end{itemize}

\paragraph{Synthetic motion robustness sweeps (label-free).}
To probe motion robustness on real IQ without ground-truth labels, we implement controlled motion-injection sweeps on Shin IQ cubes (\path{scripts/shin_ratbrain_motion_sweep.py}). In addition to simple per-frame subpixel translations, global phase modulation, and spatially smooth low-frequency clutter tones, we define a \emph{brain-like} motion model designed to be mechanistically comparable to Brain-SkullOR: a bulk rigid translation (correctable by phase-correlation registration) plus a smooth, depth-decaying elastic warp with temporally correlated coefficients (not removable by translation-only registration). Motion is applied directly to the beamformed IQ cube (a stress test rather than a full scatterer-level simulator), after which we rerun the same baseline and STAP pipeline (KA disabled to isolate STAP).

We report two label-free robustness endpoints as functions of motion amplitude: (i) PD-mode score-map correlation versus the no-motion reference (after rigid alignment and a boundary crop), and (ii) a flow-proxy hit rate at a matched background-proxy tail operating point (background-proxy tail rate $=10^{-3}$), expressed in score space as $\mathbb{P}\{S_{\mathrm{PD}}\ge\tau\mid \text{bg proxy}\}=10^{-3}$. Concretely, for each clip/window we define a fixed pixel-level flow-proxy mask from the no-motion baseline PD-derived map $pd_{\mathrm{base}}$ (top quantile; minimum pixel count) and a background-proxy mask from its complement; for each run/method we choose $\tau$ on that run's background proxy and then report the fraction of flow-proxy pixels satisfying $S_{\mathrm{PD}}\ge\tau$. As a degeneracy check (to rule out ``stability by collapse''), the sweep script also records non-collapse diagnostics (map std/entropy and background variance ratios). A post-hoc sensitivity sweep recomputes correlation under different crop margins and with alignment disabled (outputs in \path{reports/shin_motion_brainlike_batch_U_crop_sensitivity.csv}).

\paragraph{Limitations and claim boundaries (Shin).}
The Shin RatBrain Fig3 analysis is intended as an end-to-end \emph{real-IQ viability and safety} anchor, not a clinical detection benchmark. Because there are no task labels and no ground-truth vessel segmentation, the motion-sweep endpoints reported here are label-free proxies: PD-mode score-map correlation measures within-method consistency relative to each method's own no-motion reference, and the flow-proxy hit-rate endpoint measures persistence of a fixed, baseline-derived flow proxy under a matched background-proxy tail operating point. In addition, the ``brain-like'' motion stress test is an image-domain warp applied to beamformed IQ (rigid + depth-decaying elastic components) rather than a scatterer-level forward model; it is designed to mimic the specific high-rank, spatially varying residual-motion mismatch represented in Brain-SkullOR, not to reproduce full ultrasound physics. Accordingly, we interpret Shin results as evidence of (i) deterministic real-IQ ingestion and telemetry, (ii) contract v2 auditability and invariants (shrink-only; exact protected-set invariance), and (iii) self-consistency under controlled residual-motion corruption, while leaving ROC uplift claims to labeled regimes (Brain-* simulations) and to future raw-IQ datasets with task/landmark structure.

\subsection{MaceBridge Doppler simulations: Mac{\'e}-aligned bridge (stress test)}
\label{sec:macebridge}

To bridge between PD-only Mac{\'e} hemodynamic results and Doppler STAP on raw IQ, we built a Mac{\'e}-aligned Doppler simulation scaffold (\emph{MaceBridge}) that reuses the Allen atlas geometry on a small set of visually contract-positive Mac{\'e} planes and attempts to inject microvascular (Pf) and alias-like (Pa) structure in a Brain-* style Doppler regime \cite{Mace2013,Imbault2017,Urban2022,Deffieux2021,Demeulenaere2022,TCDVelocities,TCDOptimalFlow,LampeFUSBrain}. A MaceTemplate derived from the Zenodo bundle provides Mac{\'e}-native grids, Allen ROI masks (VIS/SC/LGd as H$_1$, CP/CA/DG as H$_0$), and an $8\times 8$/stride-3 tiling compatible with the fixed STAP profile used in our Brain-* simulations (code preset \texttt{--stap-profile clinical}).

In the current open-skull MaceBridge pilots, Doppler telemetry indicates that the intended Pf/Pa structure is \emph{not} realized: Pf peak occupancy in H$_1$ tiles is near zero and most tiles are Pa-dominant, and the whitened alias score $m_{\mathrm{alias}}=\log((E_a+\epsilon)/(E_f+\epsilon))$ is not class-differential (often flat or inverted). This is exactly the regime in which a contract-driven KA layer should refuse to act. Consistent with the contract posture throughout this document, we therefore treat MaceBridge as a \emph{contract-negative stress test} rather than a source of headline results: it is used to validate that telemetry-integrity checks (flat risk metric, guard/edge domination, non-actionable tails) drive the contract into C0 (KA off) with explicit reasons, and to highlight that simply matching anatomy without raw-IQ band realizability is insufficient for Doppler-side KA uplift.

Future work on MaceBridge is to replace these coarse pilots with a scatterer- or vessel-field simulation whose microvascular velocity priors in H$_1$ and alias priors in H$_0$ are calibrated to the Mac{\'e} PD contrast statistics, so that Pf/Pa telemetry becomes non-degenerate in Doppler space. Until then, MaceBridge is used only as an engineering sandbox and as a reminder of the data requirement emphasized throughout: Doppler-side STAP/KA evaluation requires raw kHz slow-time IQ with minimal upstream filtering.

\subsection{Negative real-data stress tests}
Wang ULM IQ and PICMUS carotid RF: Pf peak $\approx 0$, Po dominates, alias metrics show no class separation, and PD-z/BR/GLRT ROCs are flat under the frozen Doppler bands. Both are classified as out of regime and used only as contract-negative checks (details in \SuppOrApp{app:negative-real}).

\section{Related work}
\label{sec:related}

\textbf{STAP and knowledge-aided STAP.} Classical MVDR/LCMV/STAP and knowledge-aided variants shape covariances using external structure \cite{Capon1969Beamformer,FrostBeamformer, WardSTAPTutorial, GuerciSTAPBook2003,WicksKASTAP2006,MelvinGuerciKAParadigm2006}. Our contract formalizes when such transforms are allowed to act in fUS and keeps them off when data are out of regime.

\textbf{Ultrasound clutter suppression.} SVD clutter filters \cite{Mace2013,Rabut2020,ICDFUS2021}, robust PCA, and higher-order SVD have been used for fUS denoising. We benchmark STAP against MC--SVD, RPCA, and HOSVD baselines in brain-like regimes.

\textbf{fUS detection and hemodynamic analysis.} GLM-based detection and atlas-driven ROI analysis are common in hemodynamic fUS \cite{Demeneglmband2016,UrbanWholeBrain2015}; our hemodynamic STAP variant reuses the same banded temporal machinery on PD time series and adds alias-aware gating as a safety layer.

\section{Discussion and future work}
\label{sec:discussion}

\textbf{What the current results show.} Quantitative low-FPR detection gains are demonstrated on labeled Brain-* k-Wave simulations under a fixed STAP profile. The optional conditional execution path is reported as a compute heuristic that skips STAP outside a flow-proxy mask and falls back to the baseline there; it reduces runtime in our experiments but changes the decision rule and requires separate validation for information leakage. The KA layer is positioned as conservative risk control: it is a non-increasing score veto with explicit state/reason outputs and protected-set invariants, and it is often disabled (C0) when telemetry is flat or non-actionable. Mac{\'e} (PD-only) and Shin (unlabeled IQ) analyses are used as instrumentation anchors (telemetry integrity, contract activation rates, and label-free stability/hygiene proxies), not as evidence of detection efficacy.

\textbf{Data and regime limitations.} Public Doppler-like datasets are often post-filtered, low-PRF, or already PD-like, so Pf/Pa telemetry can be flat or guard-dominant and uplift-eligibility conditions fail. This is not a failure mode but a design outcome: the contract makes the mismatch explicit and keeps KA off. Full evaluation of Doppler-side KA in humans/NHP requires raw kHz PRF IQ pre-clutter, with known PRF/geometry, sufficient tail statistics, and task/landmark proxies to validate low-FPR behavior.

\textbf{Future work.} (i) Extend the contract dashboard to additional real PD-only datasets (fUS-BMI and cranial-window PD) and report systematic C0/C1 activation rates and ``map hygiene'' endpoints (null clusters vs hit loss). (ii) Acquire/run on in-regime labeled raw-IQ datasets (phantoms, animal, or clinical) to evaluate Doppler-side KA under a pre-registered reporting discipline. (iii) Tighten the deployment latency story with hardware-specific implementations and explore compact, frozen learned calibrators only if they preserve shrink-only and auditability guarantees.

We now instantiate the modality-agnostic contract and operating logic from the previous section for brain fUS. \subsection{k-Wave Brain fUS Simulation: Methods}
\label{sec:brain-kwave-methods}

We summarize the concrete experimental setup used for our k-Wave brain fUS simulations and the downstream STAP analysis. These details serve both as a reproducible protocol and as context for the feasibility conditions above.

\subsection{Acquisition and Simulation}

We use a k-Wave--based brain fUS simulation with parameters chosen to approximate acquisition settings reported for intra-operative and skull-window brain fUS studies \cite{Imbault2017,Rabut2020,Mace2013}:
\begin{itemize}[nosep]
  \item Center frequency $f_0 = 7.5$ MHz; speed of sound $c\approx 1540$ m/s.
  \item Per-angle PRF $=1500$ Hz; five insonification angles $(-12,-6,0,6,12)^\circ$.
  \item Five ensembles (angle sets), each with 5 angles; total compounded frames $T=320$.
  \item Spatial grid approximately $240\times 240$ pixels.
\end{itemize}

We inject several forms of motion and flow:
\begin{itemize}[nosep]
  \item \emph{Flow Doppler:} per-pixel flow Doppler uniformly in $\left[60,180\right]$ Hz (corresponding to microvascular velocities of a few--tens mm/s in the literature \cite{Imbault2017,Urban2022}).
  \item \emph{Background alias:} a fraction $0.3$ of background pixels in a depth range $[0.30,0.70]$ contaminated by a high-velocity component with center frequency $\approx 650$ Hz and $\pm 50$ Hz jitter.
  \item \emph{Aperture phase errors:} spatially correlated phase noise with standard deviation $0.8$ rad and correlation length 14 elements.
  \item \emph{Temporal clutter:} spatially extended low-frequency clutter with exponent $\beta=1$ and SNR $\approx 20$ dB over depth $[0.20,0.95]$.
\end{itemize}

Acceptance bundles export pixel-level evaluation masks \texttt{mask\_flow.npy} and \texttt{mask\_bg.npy} that define $H_1/H_0$ for ROC reporting. In our Brain-* simulations these evaluation masks are generated deterministically from the simulator's flow/background injection geometry (and the depth-zone rule that the pial band is always $H_0$), and therefore do not depend on any method output. Bundles also export a label-free PD proxy mask \texttt{mask\_flow\_pd.npy} derived from the baseline PD brightness proxy $\mathrm{PD}^{\uparrow}$; this proxy is used for conditional execution and contract telemetry. Conditional execution uses \texttt{mask\_flow\_stap\_gate.npy} (by default \texttt{mask\_flow\_pd.npy}), and in leakage ablations this gating mask is derived from either the same window, a disjoint window, or a randomized mask with matched coverage, while \texttt{mask\_flow.npy}/\texttt{mask\_bg.npy} remain fixed for evaluation. On real data, where simulator truth is unavailable, \texttt{mask\_flow.npy}/\texttt{mask\_bg.npy} are proxy-derived from a conventional PD brightness proxy $\mathrm{PD}^{\uparrow}$:
\begin{itemize}[nosep]
  \item Mode \texttt{pd\_auto}: PD threshold at quantile $0.995$ within a depth range $[0.25,0.85]$.
  \item Morphological dilation (two iterations) to fill small holes.
  \item All pixels outside the depth range are treated as background.
\end{itemize}

We generate 12 independent seeds of this scenario, each with different random draws of flow, clutter, and phase noise. Each seed gives a dataset
\[
  \texttt{pw\_7.5MHz\_5ang\_5ens\_320T\_seed}s,\quad s=1,\dots,12.
\]

A ``standard'' 320-frame open-skull brain motion-and-clutter regime (the \emph{Brain-OpenSkull} profile introduced below) is used for most acceptance and regression runs. It uses the above acquisition with flow Doppler $\in[60,180]$ Hz, background alias fraction $0.3$ in depth $[0.30,0.70]$ at 650 Hz with $\pm50$ Hz jitter, clutter over $[0.20,0.95]$, 8$\times$8 tiles, stride 3, and $L_t=8$. In the acceptance harness we typically set \texttt{--baseline svd} and \texttt{score\_mode=msd} so both $pd\_{\text{base}}$ (plain SVD high-pass) and $pd\_{\text{stap}}$ (STAP MSD) are written; PD is also saved for completeness.

A short-burst ``pial'' variant (the \emph{Brain-Pial128} profile) is also used for KA experiments. It keeps $f_0=7.5$ MHz and PRF $=1500$ Hz but shortens the sequence to $T\approx128$ compounded frames (4 ensembles $\times$ 32 pulses). Shallow alias is concentrated in depth $[0.15,0.30]$ with fraction $\approx0.7$, center $\approx650$ Hz and $\pm35$ Hz jitter, and a vibration tone at $\approx450$ Hz with surface amplitude $\approx0.35$ decaying over $\approx0.30$ depth. A weak deep alias layer is injected in $[0.35,0.75]$ with fraction $\approx0.05$ at the same center/jitter. Parenchymal flow is confined to $[0.35,0.85]$ with Doppler $\in[60,180]$ Hz and the same 8$\times$8/stride-3 tiling and $L_t=8$ STAP settings. For evaluation we enforce the pial band as background ($H_0$) for ROC; the PD-derived proxy mask used for conditional execution is computed via the same \texttt{pd\_auto} quantile rule over $[0.25,0.85]$ and also suppresses the pial band.

\subsection{Alias-Augmented Brain Profiles (Brain-AliasContract and Brain-SkullOR)}

To study KA uplift in a setting where alias structure and ROC headroom are simultaneously present, we design an alias-augmented brain family of k-Wave regimes, instantiated as the Brain-AliasContract and Brain-SkullOR profiles. These profiles reuse the 320-frame open-skull brain configuration above---same $f_0\approx 7.5$ MHz, PRF $=1500$ Hz, angle set, tiling, and $L_t=8$---but augment it with a pial-like alias layer, a sparse deep alias component, and a vibration mode, anchored in reported fUS and TCD physiology. In the simulator, these profiles correspond to the \texttt{hab\_contract} and \texttt{hab\_v3\_skull} regimes. We distinguish between:
\begin{itemize}[nosep]
  \item a \emph{near-oracle Brain-AliasContract} variant used as a stress test for the STAP math, in which the bands Pf/Pa and covariance model are extremely well matched to the simulator and the STAP PD/MSD detector approaches a near-saturated ROC ceiling on this synthetic world; and
  \item a more realistic \emph{Brain-SkullOR} variant, in which skull-like aberrations, depth-varying SNR, PRF jitter, residual motion, and hemodynamic nonstationarity are explicitly injected so that STAP operates under the kinds of mismatches encountered in transcranial and intra-operative data \cite{Rabut2020,PhaseAberrationCorrection,UltrafastUltrasoundBook,StaggeredPRFColorDoppler,FunctionalFUSBrain}.
\end{itemize}

\paragraph{Clinical/physical anchors.}
Microvascular brain flows in rodents and humans are typically in the $v\approx 3$--$20$ mm/s range for vessels that dominate functional responses \cite{Mace2011,Mace2013,Deffieux2021,Demeulenaere2022}. At $f_0\approx 7.5$ MHz and $c\approx 1540$ m/s this corresponds to Doppler content mainly in $\approx 30$--$200$ Hz, consistent with Pf. Low-frequency tissue motion (respiration, cardiac pulsation, bulk drift) produces strong content below $\approx 20$ Hz \cite{Mace2011,Mace2013,Imbault2017,Soloukey2020FUSAwake}. Large-vessel (e.g.\ MCA-like) flows reach $v\approx 0.3$--$0.8$ m/s, with TCD studies and grading criteria reporting mean velocities in the $50$--$150$ cm/s range \cite{Aaslid1982,Gao2002}. For our PRF, this corresponds to baseband Doppler in the kHz range that aliases into the Pa band \cite{Tang2021,Rabut2024}. Guided/skull-borne vibration modes with center frequencies of a few hundred Hz have also been observed in cranial ultrasonics \cite{Estrada2018}, legitimizing a Pa-adjacent vibration tone.

\paragraph{Depth zones and flow.}
We partition depth into shallow pial, upper parenchyma, deep parenchyma, and deep background zones:
\begin{itemize}[nosep]
  \item Zone A (pial/surface): depth $\in[0.12,0.28]$, always treated as $H_0$ for ROC. Contains large superficial vessels and strong alias/vibration.
  \item Zone B (upper parenchyma): depth $\in[0.28,0.45]$, mixture of parenchymal flow and background.
  \item Zone C (deep parenchyma): depth $\in[0.45,0.85]$, predominantly parenchymal flow with sparse deep alias.
  \item Zone D (deep background): depth $\in[0.85,1.0]$, background only.
\end{itemize}
Parenchymal flow in Zones B/C is simulated as in the 320-frame open-skull brain regime: microvascular velocities $v_{\text{flow}}\in[3,20]$ mm/s (projected along the beam) yielding Doppler content $f_D\approx 30$--$200$ Hz in Pf, with vascular masks and coverage fractions $c_f$ spanning $\approx 0.1$--$0.6$ so that tiles are mixed-flow rather than purely filled vessels \cite{Mace2011,Deffieux2021,Demeulenaere2022,Imbault2017,Rabut2024,Soloukey2020FUSAwake}.

\paragraph{Pial and deep alias structure.}
Zone A contains a pial alias layer: a fraction $\approx 0.7$--$0.8$ of background voxels are assigned large-vessel velocities $v\approx 0.3$--$0.8$ m/s so that, after aliasing at PRF $=1500$ Hz, their Doppler peaks concentrate around $\approx 650$ Hz with $\pm 50$ Hz jitter. This mimics aliased flow from superficial arteries and veins seen in intra-operative and skull-window fUS \cite{Imbault2017,Rabut2024,Soloukey2020FUSAwake} and in TCD studies of MCA flow \cite{Aaslid1982,Gao2002}. A shallow vibration tone at $\approx 450$ Hz with decay length $\approx 0.3$ depth adds a second coherent Pa component tied to skull/probe motion \cite{Estrada2018}.

To make alias evidence present in the parenchymal bands used for ROC, we inject a sparse deep alias component: in Zones B/C, a fraction $\approx 0.1$--$0.2$ of background voxels receive an additional high-velocity component that aliases into the same $\approx 650$ Hz band with moderate jitter. These voxels remain labeled $H_0$ but are spectrally Pa-dominant and coherent over local kernels, representing penetrating arteries, venous artifacts, or skull-reflected clutter in deep tissue \cite{Tang2021,Deffieux2021,Demeulenaere2022}.

\paragraph{Clutter and headroom.}
Clutter is modeled as in the 320-frame open-skull brain regime: spatially extended, low-frequency ($\lesssim 20$ Hz) 1/$f^\beta$ temporal clutter over depth $[0.20,0.95]$ with SNR $\approx 20$ dB, plus optional very low-frequency drift that makes registration meaningful \cite{Mace2013,Imbault2017,Soloukey2020FUSAwake}. Amplitudes of parenchymal flow, pial alias, deep alias, vibration, and clutter are tuned so that:
\begin{itemize}[nosep]
  \item Baseline PD detectors (MC--SVD, RPCA, HOSVD) remain very weak in the extreme tail (e.g.\ TPR $\approx 0.000$ at FPR $10^{-4}$ to three decimals) in both the 320-frame open-skull brain regime and the alias-augmented Brain-AliasContract profile.
  \item STAP PD/MSD achieves TPR in a shoulder region (e.g.\ $\mathrm{TPR}_{\text{STAP}}\approx 0.2$--$0.5$ at FPR $10^{-4}$--$10^{-3}$), so R--3 headroom holds and ROC is neither saturated at 0 nor 1.
  \item False positives in the low-FPR tail are dominated by Pa-heavy tiles in Zone A and the deep alias subset in Zones B/C, so the alias metric $m_{\mathrm{alias}}$ shows clear class separation between $H_0$ and $H_1$ (R--2 holds).
\end{itemize}
Thus the Brain-AliasContract profile is constructed to satisfy R--1/R--2/R--3 at clinically realistic PRF, $f_0$, and velocity ranges, while preserving the strong STAP-vs-baseline gains of the original 320-frame brain regime and making alias structure the dominant failure mode in the low-FPR tail.

In practice we observe two distinct behaviors:
\begin{itemize}[nosep]
  \item On the Brain-AliasContract profile with an aggressively tuned ``lab'' STAP configuration (dense fd-grid, strong conditioning, long effective training support per tile), scalar scores such as PD and MSD can approach near-saturated separation on the simulator, leading to ROC saturation at very low FPR. In this near-oracle regime there is essentially no headroom for KA; any KA layer behaves as a guarded regularizer rather than delivering measurable uplift.
  \item On the Brain-SkullOR profile paired with a fixed STAP preset (\texttt{--stap-profile clinical}; limited training snapshots per tile, conservative conditioning, slightly coarser Doppler grids), PD-based ROC behaves as intended: MC--SVD remains weak in the low-FPR tail, while STAP PD sits in a non-saturated shoulder regime at low FPR. Under the same configuration, our current MSD implementation---which clamps the MSD score to zero on $H_0$ tiles by construction---produces a formally perfect ROC and is therefore used as telemetry rather than as the primary detector score for contract checks.
\end{itemize}
For KA experiments intended to mimic realistic intra-operative and transcranial deployments, we therefore:
\begin{itemize}[nosep]
  \item use the Brain-AliasContract profile with the acceptance (lab) STAP configuration as a near-oracle stress test to demonstrate that, when STAP already saturates scalar ROC at low FPR, KA is neutral or guarded as predicted by the contract; and
  \item use the Brain-SkullOR profile with the fixed STAP preset (\texttt{--stap-profile clinical}) as a skull/OR realism regime to characterize how this conservative configuration behaves under skull-induced aberrations, depth-varying SNR, PRF jitter, residual motion, and hemodynamic nonstationarity \cite{LampeFUSBrain,Imbault2017,Rabut2020,UltrafastUltrasoundBook,StaggeredPRFColorDoppler,FunctionalFUSBrain,Soloukey2020FUSAwake}. In this Brain-SkullOR configuration the whitened band-ratio alias metric $m_{\mathrm{alias}}$ no longer satisfies R--2 (alias power leaks substantially into $H_1$ by design), so this regime is not used as a KA-positive benchmark; instead it is used to validate that the fixed STAP preset exhibits a realistic PD shoulder and non-oracle behavior before evaluation on real fUS data.
\end{itemize}

\paragraph{Implementation outline (Brain-SkullOR and fixed STAP preset).}
We implement the skull/OR-inspired Brain-SkullOR variant and the fixed STAP preset (\texttt{--stap-profile clinical}) as follows:
\begin{enumerate}[nosep,label=(\alph*)]
  \item \emph{Scenario scaffolding:} define simulation presets that reuse the 320-frame open-skull brain geometry (grid, $f_0$, PRF, angles, $T$, tiling) but add configurable depth zones, amplitude profiles, and temporal profiles for pial alias, deep alias, vibration, clutter, and flow \cite{LampeFUSBrain,UltrafastUltrasoundBook,FunctionalFUSBrain}.
  \item \emph{Skull/OR realism patches:} incorporate skull-like depth-varying attenuation and SNR, a two-scale aperture phase screen with realistic RMS and correlation lengths, per-ensemble PRF jitter and alias-center drift, residual motion after registration (rigid + mild nonrigid components tied to respiratory and cardiac cycles), broadened flow/alias spectra that straddle Pf/Cg/Pa, and slow hemodynamic modulation of flow amplitude over time \cite{PhaseAberrationCorrection,UltrafastUltrasoundBook,StaggeredPRFColorDoppler,FunctionalFUSBrain}.
  \item \emph{Fixed STAP preset and PD headroom:} introduce a fixed STAP preset in the replay code (\texttt{--stap-profile clinical}; tile size $8\times 8$, $L_t=8$, limited Hankel snapshots per tile, conservative diagonal loading and condition-number targets, fixed Pf/Pa bands and fd-grids, whitened band-ratio telemetry) and use it unchanged for all Brain-SkullOR and real-data experiments. For the Brain-SkullOR configuration we evaluate headroom using a fixed-FPR operating point on the PD score map ($S_{\mathrm{PD}}\ge\tau$ with $\mathbb{P}\{S_{\mathrm{PD}}\ge\tau\mid\mathrm{bg}\}=\alpha$) and require that MC--SVD remains weak at FPR $10^{-4}$--$10^{-3}$ while STAP exhibits a non-saturated shoulder without retuning STAP to the simulator.
  \item \emph{KA stress tests vs contract regimes:} keep the Brain-AliasContract configurations (alias-augmented contract brain with R--1/R--2/R--3 satisfied) as the KA-positive contract regimes where KA uplift (or neutrality) is evaluated at fixed low-FPR targets, and use the Brain-SkullOR configuration primarily to interpret how the fixed STAP preset behaves under skull/OR-like mismatches and how that behavior compares to real fUS datasets \cite{Imbault2017,Rabut2020,Soloukey2020FUSAwake,MobileFUS2024}.
\end{enumerate}

\subsection{Baselines}

Each dataset is processed with three different PD baselines:
\begin{description}[leftmargin=1.5em]
  \item[MC--SVD] Motion-compensated SVD clutter suppression:
    \begin{itemize}[nosep]
      \item We register frames via phase correlation (subpixel refinement) and then apply an SVD clutter filter on the slow-time$\times$space Casorati matrix.
      \item We use two rank-selection rules, and name them explicitly to avoid ambiguity:
        \begin{itemize}[nosep]
          \item \textbf{MC--SVD($K$):} remove a fixed number $K$ of leading slow-time modes (used mainly for diagnostics and legacy baselines).
          \item \textbf{MC--SVD($e$):} remove the smallest $K$ such that the cumulative temporal energy fraction exceeds $e$:
          \[
            \frac{\sum_{k=1}^K \sigma_k^2}{\sum_{k} \sigma_k^2} \ge e,
          \]
          where $\sigma_k$ are singular values of the registered Casorati matrix. For Brain-* we select $e$ once on a pre-committed calibration replay and then freeze it for all Brain-* results (current frozen value $e=0.90$; sweep report \path{reports/brain_mcsvd_energy_sweep.csv} / \path{reports/brain_mcsvd_energy_sweep.json}). For Shin (real IQ), clutter rank varies across clips/windows and we similarly freeze $e$ after a small label-free calibration sweep.
        \end{itemize}
      \item This corresponds to a strong, geometry-aware MC--SVD implementation in line with SVD clutter suppression used in current fUS practice \cite{Mace2013,Rabut2020,ICDFUS2021}.
    \end{itemize}
  \item[RPCA] Robust PCA baseline:
    \begin{itemize}[nosep]
      \item We apply a windowed Principal Component Pursuit (PCP) on a dimensionally reduced cube:
        temporal windows of length $80$, spatial downsampling by factor 2 via complex average pooling.
      \item Each window solves an inexact ALM PCP with truncated/randomized SVD, up to 80 inner iterations, with $\lambda$ chosen as $1/\sqrt{\max(m,n)}$ when not specified, following standard PCP formulations \cite{CandesRPCA,LinALM,HalkoRandomizedSVD}.
      \item The sparse component $S$ is treated as the flow signal; its time-averaged energy yields $pd\_base$.
    \end{itemize}
  \item[HOSVD] Tensor HOSVD baseline:
    \begin{itemize}[nosep]
      \item We treat the cube as a tensor $X\in\mathbb{C}^{T\times H\times W}$.
      \item We compute mode-0/1/2 SVDs and retain factors that capture 99\% of spectral energy along each mode, after downsampling spatially by factor 2.
      \item We project onto the multilinear low-rank subspace and define the residual $S=X-L$; PD is the time-averaged residual energy.
    \end{itemize}
\end{description}

We refer to the corresponding baselines as MC--SVD, RPCA, and HOSVD. In the fUS literature, clutter suppression baselines are typically drawn from the same families:
\begin{itemize}[nosep]
  \item Spatiotemporal SVD filters of Demen\'e, Song, Lok, and Huang, which treat the slow-time$\times$space data as a large Casorati matrix and remove a data-adaptive low-rank clutter subspace, often with randomized SVD and spatial downsampling for speed \cite{Mace2013,Rabut2020,ICDFUS2021}.
  \item Low-rank + sparse RPCA variants (PCP and robust Cauchy-RPCA) that decompose the cube into tissue clutter and microvascular flow components and can outperform SVD at the cost of heavier computation and tuning \cite{CandesRPCA,LinALM,HalkoRandomizedSVD}.
  \item Tensor decompositions such as HOSVD that exploit space$\times$time (and sometimes angle) structure to better separate clutter and flow, again with increased computational cost \cite{AdaptiveHOSVDClutter}.
\end{itemize}
Our MC--SVD baseline is thus directly comparable to the SVD clutter filters used in current fUS practice, while RPCA and HOSVD serve as ``research-grade'' low-rank baselines that we use to cross-check STAP against stronger, but less practical, alternatives.

\subsection{STAP Configuration}

For STAP, we use a fast batched MVDR/LCMV-style temporal core \cite{Capon1969Beamformer,FrostBeamformer,VanVeenBuckley1988,WardSTAPTutorial,GuerciSTAPBook2003} with robust covariance and DPSS-based band design. Unless an explicit $L_t$ is provided, $L_t$ is chosen per tile via a coherence heuristic; otherwise it is fixed (e.g.\ $L_t=8$ in the brain and pial runs). The key parameters for this brain fUS scenario are:
\begin{itemize}[nosep]
  \item Tile size $8\times 8$, stride 3; slow-time length $L_t=8$.
  \item Robust covariance estimator (Tyler/Huber) with bandwise shrinkage to a target condition number.
  \item Diagonal loading chosen automatically to ensure $\kappa(R+\lambda I)\le \kappa_{\text{target}}$.
  \item Flow band $[30,250]$ Hz, alias band centered at 575 Hz with width 175 Hz, plus guard.
  \item Multi-taper DPSS spectral design consistent with Section~\ref{sec:band-geometry}.
  \item Both PD and MSD score maps are produced; MSD is the default score in acceptance bundles, PD is also saved for completeness.
\end{itemize}

In all k-Wave experiments we distinguish between an aggressively tuned ``lab'' STAP regime (used to stress-test the math, where we allow long effective training support per tile and dense Doppler grids) and a more conservative fixed profile (the conservative reference profile; \texttt{--stap-profile clinical} in code) designed to reflect parameter choices plausible for intra-operative and transcranial brain fUS \cite{Soloukey2020FUSAwake}. In this conservative profile we limit the slow-time window to a few hundred compounded frames per PD-mode output window (comparable to the 200--400 ms windows used in current intra-operative fUS), cap the effective number of Hankel snapshots per tile, fix Pf/Pa bands and fd-grids at design time, and use more conservative conditioning (larger diagonal load and moderate condition-number targets) so that STAP remains robust under non-stationary clutter, skull-induced aberrations, and registration error rather than behaving like a near-oracle detector on the simulator. This profile is treated as a fixed operating point: we do not retune STAP hyperparameters per regime or per dataset when evaluating KA; instead, KA is layered on top of a frozen STAP configuration.

Under the original implementation, the MSD score map used in acceptance bundles was defined so that MSD was evaluated only on flow-supported tiles and clamped to zero on background ($H_0$) tiles; this made MSD ROC formally saturate (all $H_1$ scores $>0$ and all $H_0$ scores $=0$) even in difficult regimes. After relaxing this clamping, MSD becomes a meaningful contrast score on both classes, but in the conservative-profile configurations studied here its tails are not well aligned with the extremely low-FPR operating points of interest. We therefore treat MSD and band-ratio primarily as telemetry and visualization scores (for inspecting band structure, alias dominance, and tile-level transformations), and use the exported right-tail PD score map $S_{\mathrm{PD}}$ (stored as \texttt{score\_pd\_*.npy}; Notation) as the primary scalar for low-FPR reporting and R--3 headroom checks in the Brain-OpenSkull, Brain-AliasContract, Brain-SkullOR, and real-data instrumentation runs. In code, the conservative preset (\texttt{--stap-profile clinical}) also enables a PD-focused GPU fast path by default for PD scoring: the batched temporal core uses the same Hankel construction and robust covariance as the full MSD implementation, but computes per-snapshot flow-band energy and total whitened power to form band-fraction maps directly, bypassing the more expensive MSD-contrast kernel. This PD-only fast path preserves the PD ROC shoulder in all regimes we tested (Brain-OpenSkull, Brain-AliasContract, and Brain-SkullOR) while reducing wall-clock STAP latency by roughly 25--40\%, and is the configuration used for all PD-based conservative-profile experiments reported in this manuscript. The slower MSD-contrast fast path remains available for MSD- and KA-focused experiments and telemetry.

\subsection{Prospective real-data evaluation}

We do not claim clinical efficacy in this manuscript. Future work is to evaluate the fixed STAP preset (\texttt{--stap-profile clinical}) and contract v2 on labeled raw-IQ datasets under a pre-registered protocol, including conditional-execution leakage checks and best-effort baseline comparisons under a tune-once-then-freeze budget.


\subsection{Implementation Details}
\label{sec:implementation}

We briefly describe how the above algorithms are realized in code. The goal is not to reproduce software engineering details, but to make the mapping between the theoretical design and our concrete implementation fully explicit. All implementations are in Python, using NumPy for CPU code and PyTorch for GPU-accelerated linear algebra.

At a high level, the implementation consists of three logical pieces: a k-Wave simulation layer that generates motion-and-clutter brain volumes and baseline PD-derived maps, a temporal STAP layer that implements the MVDR/LCMV core, KA transforms, and band-energy telemetry on slow-time tiles, and a replay/analysis layer that loads simulated (or acquired) data, applies the chosen baseline and STAP configuration, and writes out PD-derived maps, masks, and contract telemetry for downstream ROC and contract checks.

We now summarize how each major component is implemented in algorithmic terms.

\subsection{MC--SVD Baseline}

Our MC--SVD baseline follows the standard motion-compensated SVD clutter filter used in ultrafast Doppler and fUS. The steps are:
\begin{enumerate}[nosep,label=(\alph*)]
  \item \textbf{Slow-time stacking.} For each spatial pixel $(y,x)$ and angle, we assemble a length-$T$ slow-time vector $x\_{y,x}\in\mathbb{C}^T$ from the compounded frames. We work either per-angle or per-angle-set depending on configuration; for the k-Wave brain scenario we use the full 320-frame stack.
  \item \textbf{Motion compensation.} When \texttt{--reg-enable} is set, we perform phase-correlation based registration along the slow-time dimension \cite{KuglinHinesPhaseCorr1975,ForooshSubpixelPC2002,StoneFourierSubpixel2001}:
    \begin{itemize}[nosep]
      \item A reference frame (median over time) is computed.
      \item For each frame, we estimate a subpixel shift via phase correlation (\path{reg-method=phasecorr}, \path{reg-subpixel=4}) and align the frame to the reference.
      \item Registration reduces bulk motion before SVD and improves clutter coherence.
    \end{itemize}
  \item \textbf{Matrix formation.} We vectorize all spatial pixels into a large matrix $X\in\mathbb{C}^{T\times N\_{\text{pix}}}$ with columns $x\_{y,x}$; registration is applied beforehand if enabled.
  \item \textbf{Low-rank SVD.} We compute the leading $K$ singular components via thin SVD:
  \[
    X = U\Sigma V^\mathrm{H},\qquad U\in\mathbb{C}^{T\times T}, \ \Sigma\in\mathbb{R}^{T\times N\_{\text{pix}}},\ V\in\mathbb{C}^{N\_{\text{pix}}\times N\_{\text{pix}}}.
  \]
  The clutter rank $K$ is chosen either as a fixed value (MC--SVD($K$)) or via an energy-fraction rule (MC--SVD($e$)) as described in the Baselines section; we denote by $U_K\in\mathbb{C}^{T\times K}$ the first $K$ left singular vectors retained as the clutter subspace.
  \item \textbf{Clutter subtraction.} The low-rank clutter estimate is $X\_{\text{clutter}} = U\_K U\_K^\mathrm{H} X$. We form a residual
  \[
    X_{\text{res}} = X - X_{\text{clutter}},
  \]
  and reshape $X_{\text{res}}$ back to a cube \texttt{Icube\_res} with shape $(T,H,W)$.
  \item \textbf{PD brightness map.} The MC--SVD PD brightness proxy is
  \[
    \mathrm{PD}^{\uparrow}_{\text{mcsvd}}(y,x) = \frac{1}{T}\sum\_{t=1}^T \left|X_{\text{res}}(t,y,x)\right|^2,
  \]
  implemented as a squared magnitude sum along the slow-time axis followed by normalization.
\end{enumerate}
Motion-compensated SVD is thus a global low-rank projector on the slow-time dimension, with registration providing a spatially coherent clutter subspace.
\paragraph{Acceptance harness.} For KA feasibility runs we also consider a plain SVD high-pass baseline (removing a fixed number of slow-time modes) and reuse the same STAP core on top; this diagnostic baseline is distinct from the MC--SVD/RPCA/HOSVD configurations used in the main ROC comparisons.

\subsection{RPCA Baseline}

The RPCA baseline follows the standard PCP formulation with inexact ALM and truncated/\allowbreak randomized SVD, with spatial downsampling and windowing for tractability; implementation details and the exact code paths used in our runs are summarized in \SuppOrApp{app:impl-details}.

\subsection{HOSVD Baseline}

The HOSVD baseline uses a truncated Tucker/\allowbreak HOSVD with energy-based rank caps (slow-time rank $\le 6$) on a spatially downsampled cube; reconstruction residuals form a PD brightness proxy $\mathrm{PD}^{\uparrow}$. Implementation details and the exact code paths used in our runs appear in \SuppOrApp{app:impl-details}.

\subsection{STAP Temporal Core}

The STAP temporal core operates on slow-time tiles and Hankelized snapshots; batching, robust covariance estimation, and band-energy computation are implemented on GPU with a PD-only fast path and conditional STAP. Implementation details and the exact code paths used in our runs are provided in \SuppOrApp{app:impl-details}.
\[
  \widehat R\_b = \frac{1}{K}\sum\_{k=1}^K x\_{b,k}x\_{b,k}^\mathrm{H},\qquad b=1,\dots,B,
\]
with optional robust reweighting (Tyler or Huber). Concretely:
\begin{itemize}[nosep]
  \item We compute an initial SCM.
  \item We then optionally iterate a Tyler or Huber update:
  \[
    w\_{b,k} \propto \frac{L\_t}{x\_{b,k}^\mathrm{H}R\_b^{-1}x\_{b,k}},\qquad
    R\_b \leftarrow \frac{1}{K}\sum\_{k} w\_{b,k}x\_{b,k}x\_{b,k}^\mathrm{H},
  \]
  followed by trace normalization.
\end{itemize}
All tiles in the batch share the same robust estimation logic; the code uses batched PyTorch primitives for the matrix products and solves.

\paragraph{Shrinkage and conditioning.}
To avoid overfitting and ensure well-conditioned inverses, we apply bandwise shrinkage and diagonal loading:
\begin{itemize}[nosep]
  \item For each tile we choose a scalar $\alpha\_b\in[0,1]$ such that $(1-\alpha\_b)\widehat R\_b + \alpha\_b \mu\_b I$ has condition number $\kappa\le \kappa\_{\text{target}}$ (typically 200), where $\mu\_b$ is the mean eigenvalue of $\widehat R\_b$.
  \item We then compute the minimal diagonal loading $\lambda\_b$ such that $\kappa(\widehat R\_b + \lambda\_b I)\le \kappa\_{\text{target}}$, based on eigenvalue bounds.
\end{itemize}
This ensures numerical stability for both MVDR weight computation and MSD whitening.

\paragraph{MVDR weights and temporal filtering.}
The core MVDR/LCMV logic resides in \path{pipeline.stap.lcmv}, with an entrypoint \path{lcmv_weights}. For each Doppler frequency $f\_d$ in the fd-grid, we form a steering vector $a(f\_d)\in\mathbb{C}^{L\_t}$ and solve, for each tile $b$,
\[
  w\_b(f\_d) = \frac{(\widehat R\_b + \lambda\_b I)^{-1}a(f\_d)}{a(f\_d)^\mathrm{H}(\widehat R\_b + \lambda\_b I)^{-1}a(f\_d)},
\]
implemented using batched Cholesky solves in \path{cholesky_solve_hermitian}. The weights are then applied to the Hankelized snapshots to form slow-time filtered outputs and base temporal suppression (e.g.\ temporal MVDR power).

\paragraph{Doppler grid and band projectors.}
The frequency grid $f\_d$ is constructed in \path{build_fd_grid_span}, given PRF, $L\_t$, and a relative span (e.g.\ \texttt{fd-span-rel} \texttt{0.30,1.10}). The grid is split into flow, guard, and alias bands according to the design thresholds in Section~\ref{sec:band-geometry}. We form temporal steering matrices
\[
  A(f\_d)[\ell] = e^{-2\pi i f\_d \ell / \text{PRF}},\qquad \ell=0,\dots,L\_t-1,
\]
and build projectors $\Pf,\Pa$ from these (via \path{projector_from_tones}) to realize the band geometry.

\paragraph{MSD and bandwise PD.}
After whitening with $R\_b + \lambda\_{\text{msd},b}I$, we project Hankel columns onto the whitened flow and alias subspaces and accumulate band energies:
\[
  \begin{aligned}
    E\_{f,b} &= \sum\_{f\_d\in\Cf}\sum\_{k} \bigl|a\_f(f\_d)^\mathrm{H} y\_{b,k}\bigr|^2,\\
    E\_{a,b} &= \sum\_{f\_d\in\Ca}\sum\_{k} \bigl|a\_a(f\_d)^\mathrm{H} y\_{b,k}\bigr|^2,
  \end{aligned}
\]
where $y\_{b,k}$ are whitened snapshots. The MSD-like score combines a band ratio $E\_{f,b}/E\_{a,b}$ with a contrast term, and is aggregated to per-pixel maps. In \texttt{score\_mode=pd}, acceptance bundles export the right-tail PD score maps \texttt{score\_pd\_*.npy} used for ROC reporting (and may also include legacy \texttt{pd\_*.npy}; see Notation).

\paragraph{Fast path and batching.}
The ``fast'' STAP path is enabled when KA and heavy telemetry are disabled. In this mode:
\begin{itemize}[nosep]
  \item All tiles in a frame are processed in a single batched call: Hankel construction, robust covariance estimation, shrinkage, lambda selection, whitening, MVDR weight computation, and band energy accumulation are all vectorized across the batch dimension.
  \item PyTorch is used to leverage GPU acceleration, with custom batched Cholesky and Hermitian solvers in \texttt{pipeline.gpu.linalg}.
  \item Python-level loops over tiles are avoided; only per-batch or per-frequency loops remain.
\end{itemize}
This batched GPU implementation underlies the latencies reported in Section~\ref{sec:brain-kwave-results} while matching the numerical behavior of a reference per-tile implementation (validated by comparing exported \texttt{score\_pd} maps and corresponding ROC curves).

\subsection*{Role of STAP vs KA in brain fUS}

Across all Brain-* profiles, the conservative fixed-profile STAP PD pipeline (\texttt{--stap-profile clinical})---motion-compensated SVD baseline followed by the temporal STAP core with PD-based scoring---is the primary detector and the source of the large gains over SVD/RPCA/HOSVD in the low-FPR regime. KA is a specialist, contract-driven layer that is allowed to act only when regime and gating conditions are satisfied and is otherwise neutral or explicitly disabled.

Concretely:
\begin{itemize}[nosep]
  \item The conservative fixed profile (\texttt{--stap-profile clinical}) is frozen at design time (tile geometry, $L_t$, Pf/Pa bands, conditioning targets, training support) and reused unchanged across Brain-OpenSkull, Brain-AliasContract, Brain-SkullOR, and real-data experiments. All headline ROC and latency numbers in this section compare \emph{STAP PD} against MC--SVD/RPCA/HOSVD baselines; KA is either off or acts as a shrink-only regularizer on top of STAP.
  \item In Brain-OpenSkull and Brain-SkullOR, the alias separation precondition R--2 fails or is weak, so the KA contract does not hold. In these profiles Pa-only KA has negligible or slightly negative impact on PD ROC and is treated as a guarded regularizer or risk-control layer; the regime sentinel and invariance/safety guards are configured so that KA can be disabled without affecting baseline STAP behavior.
  \item In Brain-AliasContract, R--1/R--2/R--3 and gating coverage conditions can be approximately satisfied, so KA is contract-eligible. Even there, at the ultra-low FPR operating points of interest, Pa-only covariance KA is empirically ROC-neutral on PD and is best viewed as a controlled regularizer with explicit guardrails rather than a primary source of uplift.
  \item KA contract v2 (C0/C1/C2) is implemented as a shrink-only score-space veto on top of STAP PD with protected sets and audit sentinels; its role in the simulations reported here is primarily as an auditable risk-control layer, not as a performance-boosting detector.
\end{itemize}

\subsection{Brain k-Wave Summary: STAP, Baselines, and KA}
\label{sec:brain-kwave-results}

We briefly summarize how the proposed STAP and KA configurations behave across the simulated regimes described above. The goal is not to fixate on a single numerical experiment, but to capture the qualitative regime behavior that informs our design choices.

\paragraph{STAP vs baselines.}
Across the Brain-OpenSkull, Brain-Pial128, Brain-AliasContract, and Brain-SkullOR profiles, we compare PD-based detectors built on:
\begin{itemize}[nosep]
  \item MC--SVD($e$) with an energy-fraction threshold $e$ selected once per dataset class and then frozen (motion-compensated SVD, removing the smallest number of slow-time components that account for at least an $e$ fraction of temporal energy),
  \item RPCA and HOSVD baselines tuned once per study, and
  \item the conservative fixed STAP profile (\texttt{--stap-profile clinical}; Section~\ref{sec:implementation}) applied on top of the MC--SVD residual.
\end{itemize}
In all profiles we observe:
\begin{itemize}[nosep]
  \item At very low false positive rates (FPR $10^{-4}$--$10^{-3}$), PD detectors built directly on MC--SVD, RPCA, or HOSVD residuals have very low TPR (often close to zero in the extreme tail).
  \item Under the same data, masks, and FPR targets, the conservative profile yields a substantial TPR uplift over each baseline. In the easiest Brain-OpenSkull runs, STAP PD TPR can approach a clear shoulder while baseline TPR remains near zero; in the harder Brain-AliasContract and Brain-SkullOR profiles, STAP remains clearly better than all baselines but with reduced headroom, as expected.
  \item These gains are robust across seeds and profiles: for each configuration, thresholding the exported right-tail PD score map $S_{\mathrm{PD}}$ from STAP consistently yields higher TPR than thresholding the corresponding baseline $S_{\mathrm{PD}}$ at matched low-FPR targets.
\end{itemize}

\paragraph{Profile-wise behavior.}
\begin{description}[leftmargin=1.5em,style=nextline]
  \item[Brain-OpenSkull.] In this motion-and-clutter profile the conservative-profile STAP PD detector substantially outperforms MC--SVD/\allowbreak RPCA/\allowbreak HOSVD at FPR $10^{-4}$--$10^{-3}$, with a clear shoulder in PD ROC while baselines remain near zero. Alias separation is weak (R--2 fails), so Pa-only KA is not expected to help; when enabled it behaves as a guarded regularizer and is not used for headline ROC.
  \item[Brain-Pial128.] This short-burst shallow-alias profile is used primarily to stress-test alias metrics and mask and gating design rather than as a primary ROC benchmark. The conservative-profile STAP PD configuration improves robustness to shallow alias, but KA remains a telemetry and risk-control tool and is not relied on for performance claims.
  \item[Brain-AliasContract.] In this alias-augmented contract brain profile, baseline PD detectors remain weak at low FPR and the conservative-profile STAP PD detector sits in a non-saturated shoulder regime with clear alias separation between $H_0$ and $H_1$. The KA preconditions and gating coverage can be met, but Pa-only covariance KA is empirically ROC-neutral on PD at FPR $10^{-4}$--$10^{-3}$. We therefore treat it as a controlled regularizer layered on top of STAP, not as the primary driver of uplift.
  \item[Brain-SkullOR.] In the skull-OR realism profile, MC--SVD, RPCA, and HOSVD have essentially zero TPR at very low FPR, while the conservative-profile STAP PD detector still achieves a clear shoulder, demonstrating robustness under skull-like aberrations and motion. Alias power deliberately leaks into $H_1$ so R--2 fails by design; KA is out-of-contract, and any KA layer is either disabled or used only as a conservative risk-control mechanism on top of STAP.
\end{description}

\paragraph{KA behavior across regimes.}
We also evaluate the knowledge-aided (KA) transform and band-ratio metrics:
\begin{itemize}[nosep]
  \item In the Brain-OpenSkull and Brain-SkullOR profiles, alias metrics $m_{\mathrm{alias}}$ show weak or inverted class separation (R--2 fails), and STAP PD is either already strong or operating on error modes not dominated by Pa. In these profiles, enabling Pa-only KA has negligible or slightly negative impact on PD ROC; KA behaves as a guarded regularizer and is not used to drive headline results.
  \item In the Brain-AliasContract profile, where we deliberately structure alias to live primarily in $H_0$ and tune STAP into a shoulder regime (R--3 holds), the KA contract conditions can be approximately satisfied. Even there, Pa-only KA is empirically neutral on PD at the ultra-low FPR operating points we focus on; we therefore treat KA as an optional extension rather than a primary performance driver.
  \item KA contract v2 applies a shrink-only veto factor on tiles deemed high-risk by band/coverage telemetry, reducing scores on gated tiles while preserving a protected set of high-confidence flow tiles. In the Brain-AliasContract and Brain-SkullOR simulations studied here it is approximately ROC-neutral; its primary role is as a conservative, auditable risk-control layer rather than a source of measurable uplift on the simulator.
\end{itemize}

\paragraph{Latency and conditional STAP.}
We measured end-to-end processing time for the four clutter-suppression families (MC--SVD, RPCA, HOSVD, and STAP with the conservative PD profile, \texttt{--stap-profile clinical}) on the main brain fUS profiles. Although the k-Wave pilots have $T=320$ compounded frames (5 ensembles $\times$ 64 pulses), the latency suite processes \emph{64-frame windows} extracted from each pilot (\texttt{--time-window-length 64} in \path{scripts/replay_stap_from_run.py}). For each run we report the baseline PD time (clutter filter only) and the total time when the same 64-frame window is passed through the conservative STAP PD pipeline.\footnote{Timings are taken from per-bundle runtime logs (\texttt{meta.json}) and include I/O and registration. Each bundle records \texttt{stap\_device} (\texttt{cpu} vs \texttt{cuda}); a representative hardware snapshot (GPU name, CUDA and Torch versions) is recorded by the acceptance harness under \texttt{hw} (e.g.\ \path{runs/pilot/r1_real/acceptance_r1.json/acceptance_20251101_190714.json}; note \path{acceptance_r1.json} is a directory name in this repository).}
In a first set of experiments (before conditional STAP), the conservative STAP PD profile with PD-only fast path and full 8$\times$8 tiling incurred $\approx 40$--$60\times$ the latency of MC--SVD:
\begin{itemize}[nosep]
  \item \textbf{Open-skull brain simulation.} On the Brain-OpenSkull profile (k-Wave brain, seed 1), the literature MC--SVD baseline takes $\approx 1.2$~s per 64-frame window to form baseline PD maps. Running the conservative STAP PD profile (with the PD-only fast path) on the same window costs $\approx 61$~s. Thus STAP is roughly $50$--$60\times$ slower than MC--SVD on this scenario, while improving PD TPR at low FPR from $0.000,0.000,0.001$ (MC--SVD; to three decimals) to $\approx 0.21,0.36,0.48$ at FPR $10^{-4},3\times 10^{-4},10^{-3}$ (see the brain ROC section above for details).
  \item \textbf{Alias-augmented contract brain (Brain-AliasContract profile).} On the Brain-AliasContract profile (alias-augmented k-Wave brain, seed 2), the MC--SVD baseline again runs in $\approx 1.0$~s per 64-frame window, whereas the conservative STAP PD profile takes $\approx 45$~s. Here STAP is $\approx 40\times$ slower than MC--SVD, but lifts PD TPR at low FPR from essentially zero to $\approx 0.17,0.27,0.44$ at FPR $10^{-4},3\times 10^{-4},10^{-3}$.
  \item \textbf{Skull/OR-inspired brain realism (Brain-SkullOR profile).} On the Brain-SkullOR profile (seed 2; simulator preset \texttt{hab\_v3\_skull}), MC--SVD PD runs in $\approx 1.2$~s per 64-frame window, while the conservative STAP PD profile costs $\approx 56$~s (with the PD-only fast path). In the same profile, RPCA and HOSVD baselines are themselves expensive: the RPCA baseline PD computation takes $\approx 60$~s and the HOSVD baseline $\approx 57$~s per window, both on the order of the STAP runtime. When STAP is applied on top of these baselines, the total times are $\approx 57.5$~s for RPCA+STAP and $\approx 49$~s for HOSVD+STAP. In PD ROC terms, all three baselines (MC--SVD, RPCA, HOSVD) have essentially zero true-positive rate at FPR $10^{-4}$--$10^{-3}$ on this Brain-SkullOR profile, whereas the conservative STAP PD detector achieves a shoulder around $\text{TPR}\approx 0.17,0.27,0.44$ at the same FPRs when layered on top of MC--SVD and saturates the ROC when layered on top of RPCA.
\end{itemize}
After introducing two implementation changes---a fully batched band-energy kernel in the PD fast path and a conservative \emph{conditional STAP} scheme that skips STAP on tiles with zero flow coverage in the flow mask and falls back to the baseline PD in those tiles---we obtain substantially lower latency while preserving PD ROC in all simulated profiles:
\begin{itemize}[nosep]
  \item On the Brain-OpenSkull profile (seed 1), conditional STAP skips $\approx 6{,}459$ of the $\sim 6{,}912$ tiles (those with no flow mask overlap) and reduces STAP total time from $\approx 61$~s to $\approx 4.7$~s per 64-frame window (\texttt{stap\_total\_ms} $\approx 4{,}686$), with PD TPR unchanged to three decimal places at FPR $10^{-4},3\times 10^{-4},10^{-3}$ (still $\approx 0.21,0.36,0.48$).
  \item On the Brain-AliasContract profile (alias-augmented brain, seed 2), conditional STAP skips $\approx 5{,}704$ tiles and reduces STAP total time from $\approx 45$~s to $\approx 3.9$~s (\texttt{stap\_total\_ms} $\approx 3{,}864$), while PD TPR at FPR $10^{-4},3\times 10^{-4},10^{-3}$ remains $\approx 0.17,0.27,0.44$.
  \item On the Brain-SkullOR profile (seed 2; simulator preset \texttt{hab\_v3\_skull}), conditional STAP similarly skips $\approx 5{,}704$ tiles and reduces STAP total time from $\approx 56$~s to $\approx 3.2$~s (\texttt{stap\_total\_ms} $\approx 3{,}222$), again without changing the PD shoulder at low FPR: STAP TPR remains $\approx 0.17,0.27,0.44$ at FPR $10^{-4},3\times 10^{-4},10^{-3}$ on this profile.
\end{itemize}

\paragraph{Conditional-execution leakage ablation (mask source).}

\paragraph{Implementation summary.}
Algorithmically, the fixed STAP PD path (\texttt{--stap-profile clinical}) used in our Brain-* simulation experiments can be summarized as follows:
\begin{itemize}[nosep]
  \item A motion-compensated SVD clutter filter is used as the baseline: for each window we register frames to a reference, assemble a slow-time$\times$space Casorati matrix, and remove a low-rank tissue subspace chosen by an energy-fraction rule (``literature MC--SVD''), yielding a baseline PD-derived map (and a PD brightness proxy $\mathrm{PD}^{\uparrow}$ when needed for masks/proxying) in $\approx 1$~s per 64-frame window.
  \item The fixed STAP configuration (\texttt{--stap-profile clinical}) fixes tile geometry (tiles of size $8\times 8$ with stride 3), a slow-time aperture $L_t=8$, conservative shrinkage and diagonal loading to keep temporal covariances well-conditioned, and a fixed Doppler band design (flow/alias bands and fd-grid) that is reused across regimes; a short slow-time window (roughly 200~ms) and a cap on Hankel snapshots per tile limit the effective training support.
  \item In the PD-focused fast path, the temporal STAP core builds Hankelized slow-time snapshots per tile, estimates a robust covariance for each tile, whitens the snapshots, and computes a flow-band energy fraction (matched-subspace statistic) in a fully batched way across tiles. These band fractions are then aggregated over slow time and combined with the baseline map to form the exported right-tail PD score map $S_{\mathrm{PD}}$ (stored as \texttt{score\_pd\_stap.npy}) and the legacy \texttt{pd\_stap.npy} map.
  \item Tiles are processed in GPU batches (typically on the order of a few hundred tiles per batch) to amortize Hankel construction, covariance estimation, and band-energy computation over many tiles while respecting memory limits.
  \item Conditional execution uses the flow mask to decide where STAP is applied: tiles whose support contains no flow-mask pixels are not passed through STAP at all; instead they retain the baseline output, while tiles with any flow-mask coverage receive the full STAP processing. This focuses STAP effort on the subset of tiles that can plausibly contain flow while leaving excluded tiles unchanged.
  \item The implementation records timing and coverage telemetry (number of tiles processed by STAP, number skipped, and per-stage runtimes for Hankel construction, covariance estimation, and band-energy computation) for every run; the latency and ROC summaries above are derived from this telemetry under fixed low-FPR thresholding of $S_{\mathrm{PD}}$ (Notation).
\end{itemize}

\paragraph{Robustness sweeps.}
To quantify how stable the fixed STAP PD uplift is under realistic acquisition and tiling variations, we added a robustness-sweep driver (\path{scripts/run_brain_replay_sweep.py}) and aggregator (\path{scripts/aggregate_brain_sweep.py}). The sweep spans Brain-OpenSkull and Brain-SkullOR pilots over PRF $\{1.2,1.5,2.0\}$~kHz, tile sizes $\{6\times 6,8\times 8,12\times 12\}$ with fixed strides $\{2,3,4\}$, slow-time apertures $L_t\in\{6,8,10\}$, and modest nuisance perturbations (clutter SNR offsets $\pm 3$~dB around the nominal brain SNR and alias amplitude scales $\{0.8,1.0,1.2\}$). For each configuration we reuse a k-Wave pilot per (regime, PRF), replay the fixed STAP PD profile (\texttt{--stap-profile clinical}) on top of an MC--SVD baseline using \path{scripts/replay_stap_from_run.py} (KA disabled), record per-window latency and conditional-execution coverage from \texttt{stap\_fallback\_telemetry}, and extract TPR at FPR $\{10^{-4},3\times 10^{-4},10^{-3}\}$ by thresholding $S_{\mathrm{PD}}$ at fixed low-FPR targets (Notation). The aggregator folds these runs into a matrix-style JSON/CSV table with $\Delta\mathrm{TPR}$ (STAP minus MC--SVD) and latency/coverage as functions of PRF, tile geometry, $L_t$, and nuisance scales.

Across all 243 sweep points per regime (OpenSkull and SkullOR) the MC--SVD PD baseline remains essentially flat in the extreme tail (typically $0.000$--$0.001$ to three decimals at FPR $10^{-4}$--$10^{-3}$), while the fixed STAP PD profile retains a nonzero shoulder with mean TPRs of roughly $0.19,0.24,0.38$ at FPR $10^{-4},3\times 10^{-4},10^{-3}$, respectively, and ranges $0.00$--$0.40$, $0.01$--$0.50$, and $0.03$--$0.71$ across the grid. This confirms that the PD uplift is not a single hyperparameter sweet spot but persists under moderate variation in PRF, tile geometry, $L_t$, clutter SNR, and alias amplitude.

The implementation follows this decomposition closely, with separate components for simulation, temporal STAP, and replay/analysis; the exact software layout is not essential to the methodology and is therefore omitted here. A schematic overview of this pipeline is shown in Figure~\ref{fig:stap_pipeline}.
For reproducible plots, we use an automated pipeline that regenerates the three canonical bundles (Brain-OpenSkull, Brain-AliasContract, Brain-SkullOR), recomputes PD-based contract checks, and produces latency bars and low-FPR ROC points corresponding to the summary in Table~\ref{tab:brain_latency}. The same pipeline also drives the full Brain-* suite end-to-end (pilot generation, baseline and STAP replays for MC--SVD/RPCA/HOSVD, ROC aggregation) so that the main simulated results and figures in this section can be regenerated in a single scripted run.

\section{Practical Band Design for Brain fUS (PRF 1.5--5 kHz, T 8--16)}
\label{sec:band-geometry}

We provide practical band guidelines for selecting Pf/guard/Pa with explicit physical anchors.

For brain fUS at center frequencies $f_0 \approx 6$--$15$ MHz and PRF $\approx 1.5$--$5$ kHz \cite{Mace2013,Rabut2020,Imbault2017,Urban2022,LampeFUSBrain}:

\begin{itemize}[nosep]
  \item Microvascular brain flows (arterioles/venules) in humans and rodents are typically in the range $v \approx 3$--$20$ mm/s for the vessels that dominate functional responses \cite{Imbault2017,Urban2022}. At $f_0=7.5$ MHz and $c\approx 1540$ m/s, this corresponds to Doppler shifts $f_D\approx 30$--$200$ Hz.
  \item Low--frequency tissue motion (respiration, cardiac pulsation) produces strong spectral content below $\approx 20$ Hz; this should be contained in Po and largely removed by high--pass filtering and/or SVD.
  \item Large--vessel flows (e.g.\ MCA--like arteries) reach $v\approx 0.3$--$0.8$ m/s, corresponding to $f_D$ in the kHz range \cite{TCDVelocities,TCDOptimalFlow}. Depending on PRF and angle, a portion of this energy appears in high Doppler bands and can alias into the $>$400 Hz region corresponding to Pa \cite{ICDFUS2021,MobileFUS2024}.
\end{itemize}

Thus a reasonable fixed band design for brain fUS with $T\approx 8$--$16$ and PRF $\approx 1.5$--$3$ kHz is:

\begin{itemize}[nosep]
  \item Flow band $\Cf$: $[30,250]$ Hz --- microvascular flows $\approx 3$--$20$ mm/s.
  \item Guard band $\Cg$: $[250,400]$ Hz --- protects against band--edge leakage.
  \item Alias band $\Ca$: $[400, \text{Nyquist}]$ Hz --- high Doppler where large--vessel/aliased flows and certain motion artifacts live.
\end{itemize}

In discrete terms, for $T=8$ at PRF=1.5 kHz (bin spacing $187.5$ Hz), this maps to:
\begin{itemize}[nosep]
  \item $\Pf$ on first off--DC bin (with DPSS leakage covering $\approx 30$--$250$ Hz),
  \item guard around the second bin,
  \item $\Pa$ on higher bins with bin--level guard as per BG--1/BG--2.
\end{itemize}

\section{Additional real-data stress tests}\label{app:negative-real}
\paragraph{Wang ULM IQ (awake mouse).} We load the authors' IQData files exactly as in \texttt{LoadIQ.m} (column-major reshape) and build a coarse vessel/background prior on the native $320\times 161$ grid via a simple temporal high-pass (second difference or removal of a few leading temporal singular modes), high-quantile thresholding, and accumulation; when no prior is provided we fall back to PD-mean quantiles. We then apply the frozen conservative STAP PD core (\texttt{--stap-profile clinical}; tiles $8\times 8$, stride~3, $L_t=8$--$16$, Pf/Pa/guard bands scaled to kHz PRF) on 256-frame windows across all IQ files and record Pf/Pa peak fractions, alias ratios, and PD-z ROC in the low-FPR regime. On all files and windows Pf never dominates (Pf peak fraction $\approx 0$), Pa/Po absorb most energy, and PD-z ROC remains weak; widening bands or adjusting $dt$ (0.001--0.002~s) does not change this. We interpret Wang as already heavily compounded/filtered (PD-like), with the Doppler precondition R--1 failing; it serves only as a conservative contract-negative check.

\paragraph{PICMUS carotid RF (in vivo plane-wave).} Using the publicly hosted UFF carotid cross-section data, we implement a delay-and-sum beamformer to produce an IQ/PD cube and apply the same Doppler STAP configuration scaled to its $\approx 100$ Hz PRF. Slow-time energy lies near DC, Pf/Pa never dominate, alias metrics show no class separation, and PD-z/BR/GLRT ROCs are flat. We therefore classify PICMUS as out of regime for the fixed conservative Doppler profile and use it only as a stress test demonstrating that the contract telemetry correctly withholds KA/STAP claims when Pf/Pa structure is absent.

\section{PD-only contract dashboard (hemodynamic): telemetry and aggregation}
\label{app:pdonly-dashboard}

This appendix specifies the PD-only instantiation of the contract v2 telemetry and the associated ``contract dashboard'' aggregation used for systematic plane sweeps (e.g.\ Mac{\'e}/Urban whole-brain mouse fUS). The key discipline is unchanged: the \emph{runtime} contract is label-free (proxy sets), while atlas-defined $H_1/H_0$ labels are used only \emph{offline} for descriptive reporting and are never used as contract inputs.

\subsection*{A) PD-only telemetry definitions (math)}
\paragraph{PD movie and tiling.}
Fix one coronal plane (one ``configuration'') with a PD movie
\[
P[t,y,x]\in\R_{\ge 0},\qquad t=0,\dots,T-1,
\]
sampled at rate $f_s=1/dt$ (Mac{\'e}: $f_s\approx 10$~Hz). Tile each frame into overlapping spatial tiles $\Omega_i$ of size $h\times w$ (Mac{\'e}: $8\times 8$) with stride $s$ (Mac{\'e}: $s=3$).

\paragraph{Tile PD time series.}
Define the tile-averaged PD time series
\[
p_i[t] := \frac{1}{|\Omega_i|}\sum_{(y,x)\in\Omega_i} P[t,y,x].
\]
For robustness across tiles, we optionally use a log-stabilized, mean-removed series
\[
u_i[t] := \log(p_i[t]+\epsilon_{\mathrm{pd}}),\qquad
\tilde u_i[t] := u_i[t]-\frac{1}{T}\sum_{t'=0}^{T-1} u_i[t'].
\]

\paragraph{Hankel snapshots and whitening.}
Fix a slow-time aperture $L_t$ (tens of frames in PD-only). Define $K=T-L_t+1$ and Hankel snapshots
\[
x_{i,k} :=
\begin{bmatrix}
\tilde u_i[k]\\
\tilde u_i[k+1]\\
\vdots\\
\tilde u_i[k+L_t-1]
\end{bmatrix}\in\R^{L_t},\qquad k=0,\dots,K-1.
\]
Estimate a per-tile covariance (SCM or robust variant)
\[
\wh R_i := \frac{1}{K}\sum_{k=0}^{K-1} x_{i,k}x_{i,k}\Hh,
\]
apply conditioning/diagonal loading to obtain $R_i$, and whiten
\[
y_{i,k} := R_i^{-1/2}x_{i,k}.
\]
We pool whitened snapshots to a single per-tile slow-time vector
\[
z_i := \frac{1}{K}\sum_{k=0}^{K-1} y_{i,k}\in\R^{L_t}.
\]

\paragraph{Hemodynamic band projectors and energies.}
Let $\Cf,\Cg,\Ca$ denote Pf/guard/Pa hemodynamic bands (Mac{\'e}: $\Cf=[0,0.5]$~Hz, $\Cg=(0.5,1.0]$~Hz, $\Ca=(1.0,3.0]$~Hz). Build fixed temporal projectors $(\Pf,\Pg,\Pa)$ onto the span of discrete Fourier tones whose frequencies lie in these bands, and define $\Po := I-\Pf-\Pg-\Pa$ (``other''/out-of-band). Define band energies
\[
E_f(i) := z_i\Hh \Pf z_i,\quad
E_g(i) := z_i\Hh \Pg z_i,\quad
E_a(i) := z_i\Hh \Pa z_i,\quad
E_o(i) := z_i\Hh \Po z_i,
\]
and $E_{\mathrm{tot}}(i) := E_f+E_g+E_a+E_o+\epsilon$.

\paragraph{Risk metrics and Pf-peak telemetry.}
We use the same contract telemetry primitives as in the Doppler regime:
\[
m_{\mathrm{alias}}(i) := \log\frac{E_a(i)+\epsilon}{E_f(i)+\epsilon},\qquad
m_{\mathrm{guard}}(i) := \log\frac{E_g(i)+\epsilon}{E_f(i)+\epsilon},\qquad
r_g(i) := \frac{E_g(i)}{E_{\mathrm{tot}}(i)}.
\]
Define a whitened periodogram of the pooled vector $z_i$,
\[
\wh z_i[n] := \sum_{\ell=0}^{L_t-1} z_i[\ell]\,e^{-2\pi i n\ell/L_t},\qquad
S_i[n] := |\wh z_i[n]|^2,\qquad n=0,\dots,\lfloor L_t/2\rfloor,
\]
and the (non-DC) peak frequency $f_{\mathrm{peak}}(i)=\frac{n_{\mathrm{peak}}}{L_t dt}$ where
$n_{\mathrm{peak}}(i):=\arg\max_{n\ge 1} S_i[n]$. The Pf-peak indicator and occupancy on a set $\mathcal{T}$ are
\[
\pi_f(i) := \mathbf{1}\{f_{\mathrm{peak}}(i)\in\Cf\},\qquad
\mathrm{PfPeakFrac}(\mathcal{T}) := \frac{1}{|\mathcal{T}|}\sum_{i\in\mathcal{T}}\pi_f(i).
\]

\subsection*{B) PD-only proxy sets and contract inputs (math)}
The PD-only contract uses label-free proxy sets based on a PD-derived flow-coverage proxy, analogous to the IQ flow-mask coverage $c_f$.
Let $\bar P[y,x] := \frac{1}{T}\sum_t P[t,y,x]$ be the mean PD image (optionally restricted to a brain mask). Define a high-quantile flow proxy mask
\[
M_{\mathrm{flow}}(y,x) := \mathbf{1}\{\bar P[y,x]\ge Q_{q_{\mathrm{flow}}}(\bar P)\},
\]
and per-tile flow coverage
\[
c_f(i) := \frac{1}{|\Omega_i|}\sum_{(y,x)\in\Omega_i} M_{\mathrm{flow}}(y,x)\in[0,1].
\]
With frozen design thresholds $c_{\mathrm{bg}}$ and $c_{\mathrm{flow}}$ (typical $c_{\mathrm{bg}}\approx 0.05$, $c_{\mathrm{flow}}\approx 0.20$), define
\[
\mathcal{T}_{\mathrm{bg}} := \{i:\ c_f(i)\le c_{\mathrm{bg}}\},\quad
\mathcal{T}_{\mathrm{flow}} := \{i:\ c_f(i)\ge c_{\mathrm{flow}}\},\quad
\mathcal{T}_{\mathrm{nonbg}} := \{i:\ c_f(i)>c_{\mathrm{bg}}\}.
\]
If $\mathcal{T}_{\mathrm{flow}}$ is too small (insufficient proxy support), a conservative fallback is to define proxy sets via tilewise mean-$\bar P$ quantiles and record an explicit ``proxy\_fallback'' reason code.

\subsection*{C) Contract decision metrics for PD-only (math)}
\paragraph{Baseline score for PD-only contract.}
For Mac{\'e}, the baseline scalar detector is PD-z on tile time series: choose a baseline segment $B$ (e.g.\ first third of frames) and define
\[
s_{\mathrm{base}}(i) := \max_{t\notin B} \frac{u_i[t]-\mu_i^{(B)}}{\sigma_i^{(B)}+\epsilon_z},
\]
with $\mu_i^{(B)},\sigma_i^{(B)}$ the mean and standard deviation of $u_i[t]$ over $t\in B$. Higher $s_{\mathrm{base}}$ means more activation.

\paragraph{Risk-mode selection and integrity checks.}
The PD-only contract follows the same posture as the IQ contract:
if risk telemetry is flat (e.g.\ $\mathrm{IQR}(m_{\mathrm{alias}}\mid\mathcal{T}_{\mathrm{bg}})$ too small) or proxy support is insufficient, the contract returns C0 with an explicit reason (``alias\_metric\_flat'', ``insufficient\_bg\_samples'', etc.). If guard dominates (e.g.\ $Q_{0.90}(r_g\mid\mathcal{T}_{\mathrm{bg}})$ above a frozen threshold), uplift is vetoed and the contract either switches to a guard-risk safety lane (C1 only, requiring tail association of $r_g$) or returns C0 as out of regime (``guard\_extreme\_out\_of\_regime'').

\paragraph{Actionability checks (C2 eligibility, label-free).}
C2 (uplift-eligible; reporting-only) requires additional evidence that risk telemetry explains high-score background tails. In the selected risk mode ($m_{\mathrm{risk}}=m_{\mathrm{alias}}$ or $m_{\mathrm{guard}}$), define
\[
\Delta_{bf} := \operatorname{median}(m_{\mathrm{risk}}\mid\mathcal{T}_{\mathrm{bg}})-
               \operatorname{median}(m_{\mathrm{risk}}\mid\mathcal{T}_{\mathrm{flow}}),
\]
and within $\mathcal{T}_{\mathrm{bg}}$ define a tail subset $\mathcal{T}_{\mathrm{tail}}$ as the top $1\%$ of $s_{\mathrm{base}}$ and a mid subset $\mathcal{T}_{\mathrm{mid}}$ as a central quantile band; require
\[
\Delta_{\mathrm{tail}} := \operatorname{median}(m_{\mathrm{risk}}\mid\mathcal{T}_{\mathrm{tail}})-
                          \operatorname{median}(m_{\mathrm{risk}}\mid\mathcal{T}_{\mathrm{mid}})
                          \ge \Delta_{\mathrm{tail},\min}.
\]
Finally, we enforce a Pf-realization guardrail for C2:
\[
\mathrm{PfPeakFrac}(\mathcal{T}_{\mathrm{flow}})\ge p_{f,\min}.
\]
If these checks fail, the contract remains C0 or C1 (safety-only) but does not declare uplift eligibility.

\paragraph{Coverage sentinels.}
As in the Doppler contract, gating must be neither inert nor global: after defining protected/candidate sets (below), the gated fraction
\[
p_{\mathrm{shrink}} := \mathbb{P}\{g=1\mid \mathcal{T}_{\mathrm{cand}}\}
\]
must lie in frozen bounds (e.g.\ $p_{\min}\le p_{\mathrm{shrink}}\le p_{\max}$), otherwise the contract returns C0 with reason ``coverage\_out\_of\_bounds''.

\subsection*{D) PD-only shrink-only KA mapping (optional; dashboard parity)}
The PD-only instantiation supports both the hard veto used in the Mac{\'e} systematic-sweep reporting and a soft shrink mapping for dashboard parity.
Define a protected set $\mathcal{T}_{\mathrm{prot}}$ (tiles with high flow coverage and/or very high baseline scores) and a candidate set $\mathcal{T}_{\mathrm{cand}}$ (non-protected tiles in an uncertain score window). Using a background-calibrated risk threshold $\tau_{\mathrm{risk}}=Q_{q_{\mathrm{risk}}}(m_{\mathrm{risk}}\mid\mathcal{T}_{\mathrm{bg}})$, define the gate
\[
g(i)=\mathbf{1}\{i\in\mathcal{T}_{\mathrm{cand}},\ m_{\mathrm{risk}}(i)\ge\tau_{\mathrm{risk}}\}.
\]
\emph{Hard veto (Mac{\'e} sweep):} apply the binary gate $g$ as a veto on positive activations,
\[
s_{\mathrm{KA}}(i)=s_{\mathrm{base}}(i) - g(i)\max(s_{\mathrm{base}}(i),0),
\]
which is equivalent to setting positive gated tiles to zero while leaving negative scores unchanged. \emph{Soft shrink (dashboard parity):} set $w(i)=\max(w_{\min},\exp(-k\,u(i)))$ with a robust normalized exceedance $u(i)\ge 0$ and apply $s_{\mathrm{KA}}(i)=s_{\mathrm{base}}(i)-(1-w(i))\max(s_{\mathrm{base}}(i),0)$. In both cases, $g(i)=0$ (and $w(i)=1$) on $\mathcal{T}_{\mathrm{prot}}$ enforces the protected-set invariance and $s_{\mathrm{KA}}(i)\le s_{\mathrm{base}}(i)$ enforces shrink-only behavior.

\subsection*{E) Dashboard schema and aggregation (PD-only)}
For PD-only datasets the dashboard is computed per plane (configuration) and written as a CSV with one row per (scan, plane). Each row records: (i) identifiers and frozen settings (scan/plane, $T$, $f_s$, $L_t$, tile geometry, band edges), (ii) proxy support counts ($|\mathcal{T}_{\mathrm{bg}}|$, $|\mathcal{T}_{\mathrm{flow}}|$), (iii) contract outputs (state $\in\{\mathrm{C0},\mathrm{C1},\mathrm{C2}\}$, reason code, risk\_mode), (iv) key telemetry ($\mathrm{PfPeakFrac}(\mathcal{T}_{\mathrm{flow}})$, $Q_{0.90}(r_g\mid\mathcal{T}_{\mathrm{bg}})$, $\mathrm{IQR}(m_{\mathrm{risk}}\mid\mathcal{T}_{\mathrm{bg}})$, $\Delta_{bf}$, $\Delta_{\mathrm{tail}}$), and (v) KA coverage/strength when active ($p_{\mathrm{shrink}}$ and scale/shrink quantiles). Atlas-labeled descriptive fields are prefixed \texttt{offline\_} (e.g.\ \texttt{offline\_pf\_peak\_frac\_H1}, \texttt{offline\_pf\_peak\_frac\_H0}, \texttt{offline\_hits\_pre/post}) to make clear that they are not contract inputs.

Dashboard aggregation mirrors the IQ dashboard: histograms of contract states and (state, reason) pairs, a risk-mode histogram, distributions of key telemetry (Pf peak fraction, guard q90, risk IQR), and (for active planes) shrink coverage/strength summaries. Any atlas-labeled Mac{\'e} plots (e.g.\ PfPeak(H$_1$) vs PfPeak(H$_0$), alias separation) are presented as a separate, explicitly offline/descriptive panel.

\subsection*{F) ``Do not overclaim'' messaging (PD-only dashboard)}
When reporting a PD-only contract dashboard, we use the following posture in captions and limitations:
\begin{itemize}[nosep]
  \item PD-only telemetry operates on low-rate PD time series and is a contract-sentinel/auditability check; it does not validate Doppler-side uplift on raw IQ.
  \item C0 (KA off) is an intended success state when telemetry is flat or non-actionable; the dashboard is designed to make these cases explicit via reason codes.
  \item Atlas-defined $H_1/H_0$ results on Mac{\'e} are descriptive; contract-positive slices identified by scanning the same dataset are not treated as held-out confirmation.
  \item C2 is a reporting discipline requiring additional label-free actionability evidence; C1 is safety-only and should not be presented as a clinical performance guarantee.
\end{itemize}

\section{Algorithmic details (supplementary)}
\label{app:alg-details}

\subsection{KA Contract v2 state machine (code defaults)}
The contract v2 state machine is implemented in \path{pipeline/stap/ka_contract_v2.py} as \path{evaluate_ka_contract_v2}. It consumes per-tile arrays
$\{S_{\mathrm{base}},m_{\mathrm{alias}},r_g,c_f,\mathrm{PfPeak}\}$ and returns a JSON-serializable report with fields \texttt{state}, \texttt{reason}, and a \texttt{metrics} dictionary. The frozen defaults used throughout this repository are the \path{KaContractV2Config} dataclass defaults:
\begin{itemize}[nosep]
  \item \textbf{Proxy thresholds:} $c_{\mathrm{bg}}=0.05$, $c_{\mathrm{flow}}=0.20$.
  \item \textbf{Candidate/protected score quantiles:} $q_{\mathrm{lo}}=0.30$, $q_{\mathrm{hi}}=0.995$.
  \item \textbf{Risk threshold:} $q_{\mathrm{risk}}=0.90$ applied to $m_{\mathrm{alias}}$ on $\mathcal{T}_{\mathrm{bg}}$.
  \item \textbf{Telemetry integrity (flatness + guard):} $\mathrm{IQR}(m_{\mathrm{alias}}\mid\mathcal{T}_{\mathrm{bg}})\ge 0.25$; uplift veto if $Q_{0.90}(r_g)>0.25$; ``guard safety'' lane allowed up to $Q_{0.90}(r_g)\le 0.70$ if guard is actionable, requiring $\mathrm{IQR}(r_g\mid\mathcal{T}_{\mathrm{bg}})\ge 0.03$ and $\Delta_{\mathrm{tail,guard}}\ge 0.01$.
  \item \textbf{C2 actionability thresholds:} $\Delta_{bf}\ge 0.30$ and $\Delta_{\mathrm{tail}}\ge 0.20$ (computed on $\mathcal{T}_{\mathrm{bg}}/\mathcal{T}_{\mathrm{flow}}$ proxies as in Section~\ref{sec:ka-contract-v2}).
  \item \textbf{C2 Pf-realization guardrail:} $\mathrm{PfPeakFrac}(\mathcal{T}_{\mathrm{flow}})\ge 0.05$ with $|\mathcal{T}_{\mathrm{flow}}|\ge 50$.
  \item \textbf{Coverage bounds:} safety lane $p_{\mathrm{shrink}}\in[0.02,0.25]$ (guard lane $[0.02,0.15]$), uplift lane $p_{\mathrm{shrink}}\in[0.05,0.40]$.
  \item \textbf{Invariance sentinel:} if $\mathrm{IQR}(\log w\mid \text{gated}) < 0.02$ then return C0 with \texttt{invariance\_no\_effect}.
  \item \textbf{Sample support:} $n_{\min}=500$ valid tiles, $n_{\mathrm{bg},\min}=200$ background-proxy tiles, $n_{\mathrm{cand},\min}=200$ candidate tiles, $n_{\mathrm{tail},\min}=20$, $n_{\mathrm{mid},\min}=20$.
\end{itemize}
The state machine returns \texttt{C0\_OFF}, \texttt{C1\_SAFETY}, or \texttt{C2\_UPLIFT}. Disable reasons are explicit and frozen in code; common ones include:
\texttt{missing\_inputs}, \texttt{shape\_mismatch}, \texttt{insufficient\_samples}, \texttt{insufficient\_bg\_samples},
\texttt{alias\_metric\_flat}, \texttt{guard\_metric\_missing}, \texttt{guard\_extreme\_out\_of\_regime},
\texttt{guard\_dominates\_not\_actionable}, \texttt{pf\_peak\_flow\_below\_min}, \texttt{too\_few\_candidates},
\texttt{coverage\_out\_of\_bounds}, and \texttt{invariance\_no\_effect}. Successful enablement returns \texttt{reason=ok}.

\subsection{Score-space shrink mapping and exported maps}
When score-space KA is enabled, per-tile shrink factors $w(i)\in(0,1]$ are derived from the contract report by \path{derive_score_shrink_v2_tile_scales} in \path{pipeline/stap/ka_contract_v2.py} and applied in score space as
\[
S_{\mathrm{post}}(i)=w(i)\,S_{\mathrm{pre}}(i),\qquad w(i)\in(0,1].
\]
For convenience (and compatibility with legacy \texttt{pd} arrays), acceptance bundles store an equivalent per-pixel \emph{scale} map $\gamma(i)=1/w(i)\ge 1$ as \path{ka_scale_map.npy}, averaged over overlapping tiles and applied only on the union of gated tiles as
\[
\mathrm{PD}_{\mathrm{post}}(i)=\gamma(i)\,\mathrm{PD}_{\mathrm{pre}}(i).
\]
A binary \path{ka_gate_map.npy} stores the gated region. Protected pixels are enforced by setting $\gamma=1$ on the protected set (flow mask and extreme-score pixels as in Section~\ref{sec:ka-contract-v2}).

\paragraph{Implementation score conventions.}
Throughout the manuscript we report low-FPR operating points using the right-tail PD score map $S_{\mathrm{PD}}$ (stored as \texttt{score\_pd\_*.npy} when available; Notation): choose $\tau$ so that $\mathbb{P}\{S_{\mathrm{PD}}\ge\tau\mid \mathrm{bg}\}=\alpha$ and evaluate $\mathbb{P}\{S_{\mathrm{PD}}\ge\tau\mid \mathrm{flow}\}$. Under the exported map convention above, \path{ka_scale_map.npy} stores $\gamma(i)=1/w(i)\ge 1$ and is applied to the legacy \texttt{pd} map as $\mathrm{PD}_{\mathrm{post}}=\gamma\,\mathrm{PD}_{\mathrm{pre}}$ on gated regions; since $S_{\mathrm{PD}}=-\texttt{pd}$, this is equivalent to $S_{\mathrm{post}}=w\,S_{\mathrm{pre}}$ and therefore is non-increasing in score space.

\section{Implementation details (supplementary)}
\label{app:impl-details}

\subsection{RPCA baseline (full)}
The RPCA baseline used in this repository is implemented as \path{sim/kwave/common.py:_baseline_pd_rpca}. It performs a windowed Principal Component Pursuit (PCP) using an inexact augmented Lagrangian method (IALM) on a downsampled Casorati matrix:
\begin{itemize}[nosep]
  \item Spatial downsampling: complex average pooling by factor 2 when $\max(H,W)>128$.
  \item Temporal windows: default subwindow length $t_{\mathrm{sub}}=80$ (or shorter if $T<80$); windows are processed sequentially.
  \item Optimization: IALM with at most 80 inner iterations per window (\texttt{max\_iters\_inner=min(max\_iters,80)}), tolerance $10^{-4}$.
  \item Shrinkage: $\lambda = 1/\sqrt{\max(m,n)}$ when not supplied, where $m=T_w$ and $n=H_dW_d$.
  \item Low-rank step: singular value thresholding using either full SVD or a randomized SVD depending on matrix size, with a per-window rank heuristic $k=\min(8,\max(2,\lfloor T_w/10\rfloor))$.
  \item Output PD: sum of sparse-component energy over windows on the downsampled grid, then nearest-neighbor upsampling to the original resolution.
\end{itemize}

\subsection{HOSVD baseline (full)}
The tensor HOSVD baseline is implemented as \path{sim/kwave/common.py:_baseline_pd_hosvd}. The cube $X\in\mathbb{C}^{T\times H\times W}$ is optionally spatially downsampled (complex average pooling). Mode-0/1/2 SVDs are computed on the unfoldings and ranks are chosen either explicitly (\texttt{--hosvd-ranks}) or by energy fractions (\texttt{--hosvd-energy-fracs}, default $(0.99,0.99,0.99)$). The low-rank approximation is reconstructed by projecting onto the multilinear subspace, and PD is the time-averaged residual energy $\frac{1}{T}\sum_t |X(t)-L(t)|^2$. In the brain baselines used in this repository, the Makefile target \texttt{r4c-replay-hosvd} runs HOSVD with \texttt{--hosvd-spatial-downsample 2} and \texttt{--hosvd-energy-fracs 0.99,0.99,0.99}.

\subsection{STAP temporal core (full)}
For k-Wave pilots and replay bundles, the temporal STAP implementation used for acceptance bundles lives in \path{sim/kwave/common.py} (\path{_stap_pd_tile_lcmv} and helpers) and uses the same conceptual decomposition as the main text: Hankelize slow-time tiles, estimate per-tile covariances with a robust estimator (\texttt{tyler\_pca} or Huber), apply diagonal loading with condition-number targets (MVDR load in \texttt{auto} mode), whiten, and compute band-energy statistics using a fixed Doppler tone grid. The conservative-profile PD fast path and conditional STAP are enabled by environment variables (\texttt{STAP\_FAST\_PD\_ONLY=1}, \texttt{STAP\_MAX\_SNAPSHOTS}, \texttt{STAP\_SNAPSHOT\_STRIDE}) and by skipping tiles with zero flow-mask overlap (telemetry key \texttt{stap\_tiles\_skipped\_flow0} in \texttt{meta.json}). For reproducible run settings, the Makefile targets and scripts in \path{scripts/} record the exact STAP arguments passed to \path{scripts/replay_stap_from_run.py} (e.g.\ \texttt{diag\_load}, \texttt{mvdr\_auto\_kappa}, \texttt{grid\_step\_rel}, \texttt{max\_pts}) and export them in each bundle's \texttt{meta.json}.

\subsection{Covariance blending constraints and KA modes}

We permit anisotropic shrinkage, but only \emph{within} bands. In practice we consider two KA paths:
\begin{itemize}[nosep]
  \item \textbf{Updated (commuting band--scalar).} $Q\Hh Q$ is forced to be bandwise scalar with target gains $(s_f,s_a,s_o)$ (in code: $s_f=s_o\approx 1$, $s_a\approx 1.5$), small mixing, and no off--block priors; the content of $R_0$ beyond $(\Pf,\Pa)$ is ignored.
  \item \textbf{Blend.} A trace--normalized blend $R_\beta = (1-\beta) R_t^{(n)} + \beta R_0^{(n)}$ with a small diagonal lift. In practice $R_0$ is overridden to a band--scalar form (Pf weight 1, Pa weight $\gamma_a>1$, Po weight 1) so the geometry is explicit; $\beta$ is treated as a fixed hyperparameter per regime.
\end{itemize}
Both modes enforce bandwise structure; there are no off--block priors.

\subsection{CB--1: Band--Block Blending}

We construct
\[
R_\beta := (1-\beta)\,\wh R + \beta \bigl(R_{0,f}\oplus R_{0,a}\oplus R_{0,o}\bigr) + \delta I,
\]
where $R_{0,f} = \Pf R_0 \Pf$, etc., for some prior $R_0$. There are no off--block priors; $\beta\in[0,1)$ and $\delta\ge 0$.

Interpretation: this structure makes $Q$ nearly block--diagonal in $(\Po,\Pf,\Pg,\Pa)$ while still allowing bandwise control to realize OC--2.

\subsection{CB--2: Bandwise Spectral Targets}

We choose $(\beta,\delta)$ to enforce OC--2 explicitly:
\[
\lambda_{\min}(Q\Hh Q|_{\Pf}) \in [s_f^2(1-\delta_f), s_f^2(1+\delta_f)],\quad\text{etc.}
\]
with design tolerances $\delta_\bullet\le 0.1$ and separation $\Lambda = s_a/s_f \ge 1.25$.

\subsection{CB--3: Bandwise Condition Number Safeguard}

We constrain bandwise conditioning:
\[
\kappa(R_\beta|_{\Pf}) \le \kappa_f^{\max},\qquad
\kappa(R_\beta|_{\Pa}) \le \kappa_a^{\max},
\]
avoid an incompatibility between conditioning and preserving flow Rayleigh quotients.

\subsection{Gating (localizing KA in score space)}
\label{app:gating}

A global band--constant transform with $Q$ cannot improve ROC if applied everywhere (it is close to a scalar reweighting of BR/GLRT). To create re--ranking, KA must be \emph{localized} to tiles where:
\begin{itemize}[nosep]
  \item scores are dominated by alias/undesired components,
  \item positives are unlikely to lie.
\end{itemize}

We define gating based on a combination of alias metrics, flow coverage, depth, PD, and registration. This supplements G--1/G--2 with a generalized alias score $m_{\mathrm{alias}}$ from Section~\ref{sec:regime}.

\subsection{G--1: Alias--dominant tile selection}

Given a baseline tile $z$, compute per--band energies $(E_o,E_f,E_g,E_a)$ and a generalized alias score $m_{\mathrm{alias}}(z)$ satisfying R--2.

\paragraph{Data--adaptive threshold.}
For BG tiles (or their proxy), collect $m_{\mathrm{alias}}(z)$ and define
\[
\tau_{\mathrm{alias}} := Q_{q_{\mathrm{alias}}}\bigl(m_{\mathrm{alias}} \mid H_0\bigr),
\]
where $Q_q$ is the $q$--quantile and $q_{\mathrm{alias}}\in[0.7,0.9]$ is a design choice (typical: $0.8$ or $0.9$).

Define an alias gate
\[
g_{\mathrm{alias}}(z) = \mathbf{1}\left\{ m_{\mathrm{alias}}(z) \ge \tau_{\mathrm{alias}} \right\}.
\]

\paragraph{Band--edge purity.}
Additionally require that guard energy is small relative to Pf+Pa:
\[
\frac{E_g}{E_o + E_f + E_a + E_g} \le \eta_{\text{edge}},\quad
\eta_{\text{edge}}\le 0.1.
\]

Interpretation: this gate selects the top tail of alias--heavy tiles among negatives, while protecting against band--edge contamination. The distribution of $m_{\mathrm{alias}}$ across classes is governed by R--2; empirical validation (Section~\ref{sec:regime}) must show a nontrivial difference between $H_0$ and $H_1$.

\subsection{G--2: Flow coherence veto}

We can optionally compute a flow coherence metric $\mathrm{FPC}$ (first principal component fraction) within the flow band:
\[
\mathrm{FPC} := \frac{\lambda_1(\wh S_f)}{\tr(\wh S_f)},\qquad
\wh S_f := \frac{1}{K}\sum_{k=1}^K u_k u_k\Hh,\quad
u_k := \Pf z^{(k)},
\]
where $z^{(k)}$ are projected slow--time vectors across DPSS tapers or within a local ensemble.

Gate \emph{may} require
\[
\mathrm{FPC}\le c_{\text{pc}},\qquad c_{\text{pc}}\in[0.4,0.6].
\]

Interpretation: flow tiles tend to exhibit higher coherence in Pf; when enabled, we veto gating when $\mathrm{FPC}$ is large, thereby reducing positive gating probability $p_+$. Alias/motion tiles have lower flow coherence and pass this condition. In the current acceptance runs this check is typically left off; coverage and depth gating carry most of the class separation.

\subsection{G--2': Flow coverage veto}

Let $c_f$ be the tile--wise flow coverage fraction:
\[
c_f := \frac{\#\{\text{pixels labeled as flow}\}}{\#\{\text{pixels in tile}\}},
\]
using a label--free proxy such as a PD--derived flow mask.

We require
\[
c_f \le c_{f,\max},\qquad c_{f,\max}\in[0.1,0.2],
\]
and optionally $c_f \ge c_{f,\min}$ to avoid pure background noise tiles (typical $c_{f,\min}\approx 0$).

Interpretation: we prefer gating on tiles with \emph{low to moderate} flow coverage. This is the regime where alias/motion dominates and where down--scaling scores is most beneficial. Dense flow tiles are either left to STAP alone or are guarded via the flow coherence veto. (In the current implementation, coverage is treated as an \emph{upper} bound; missing coverage defaults to ``pass'' in this test.)

\subsection{G--3: Baseline score quantile window}

Let $S_{\text{base}}(z)$ be the baseline BR/GLRT score. Within a given depth stratum, compute score quantiles $q_{0.3},q_{0.95}$ over BG tiles. Gate requires
\[
S_{\text{base}}(z)\in [q_{0.3},q_{0.95}].
\]

Interpretation: avoid gating on the noise floor (very low scores) and on already high--confidence positives (very high scores). This focuses KA on the part of the negative tail that matters near the target FPR.

\subsection{G--4: Registration quality (optional)}

If motion compensation / registration is used, include a registration quality metric (e.g.\ PSR) and require
\[
\mathrm{PSR}_{\text{reg}}\le \text{PSR}_{\max},
\]
or any similar condition ensuring that tiles with gross registration failure are handled separately.

\subsection{Offline validation of gating}

On a held--out labeled configuration, define
\[
p_+ := \mathbb{P}\{g=1\mid H_1\},\qquad
p_- := \mathbb{P}\{g=1\mid H_0\},
\]
for the combined gate $g$ that incorporates G--1, G--2, G--2', G--3, G--4. Require:
\[
p_-\ge 0.2,\quad p_- \le 0.5,\quad p_+ \le 0.1 p_-.
\]

Interpretation: a nontrivial fraction of negatives are gated (but not too many), while positives are rarely gated. This condition is central to the SL--2 quantile shift argument.

\subsection{Summary: Compact Feasibility Checklist (Updated)}

KA--STAP can be expected to outperform MC--SVD (or STAP--only) in the low-FPR regime if and only if:

\begin{enumerate}[label=\arabic*.,nosep]
  \item \textbf{Regime preconditions.} Band occupancy R--1, alias separation R--2, and baseline headroom R--3 are satisfied on a held--out validation set.
  \item \textbf{Band design.} BG--1/BG--2/BG--3 hold for the chosen PRF/$T$/DPSS; Pf/Pa are physically meaningful (e.g.\ Pf $\approx$ 30--250 Hz, Pa $\gtrsim$ 400 Hz in brain fUS).
  \item \textbf{Operator geometry.} OC--1 with $\varepsilon_{\text{mix}}\le 0.05$ and OC--2 with $\Lambda = s_a/s_f \ge 1.25$ and $\delta_\bullet\le 0.1$.
  \item \textbf{Blending realizable.} CB--1/CB--2/CB--3 satisfied with chosen $(\beta,\delta)$.
  \item \textbf{Gating conservative and class--differential.} G--1/G--2/G--2'/G--3/G--4 implemented; offline, $p_-\in[0.2,0.5]$ and $p_+\le 0.1p_-$.
  \item \textbf{Regime sentinel.} RS--1 does not trigger (KA not out of regime).
  \item \textbf{Invariance sentinel.} $\wh s_a/\wh s_f \notin[1-\epsilon,1+\epsilon]$ with $\epsilon\le 0.02$; otherwise KA is effectively scalar and can be skipped.
  \item \textbf{Safety guard.} Safety--1 (monotonicity on gated flow tiles) passes.
  \item \textbf{Evaluation discipline.} GLRT controls and tail CI practices (Section~\ref{sec:eval}) in place for fair low--FPR comparison.
\end{enumerate}

When all of the above hold, KA applies a non--mixing, alias--inflating band transform chiefly to alias--dominant tiles, pushing negative scores down while largely leaving positives unchanged. This lowers the negative $\alpha$--quantile and increases TPR at fixed small FPR in a manner consistent with the SL--2 quantile shift analysis.
