# Security Policy

## Supported Versions

Security fixes are applied to the latest `main` branch state intended for active development and release preparation.

## Reporting a Vulnerability

Please do **not** open public issues for security-sensitive reports.

Use one of the following channels:
- GitHub private vulnerability reporting (preferred), if enabled for the repository.
- Email: `arthur.shune@gmail.com`

When reporting, include:
- A clear description of the issue and impact.
- Steps to reproduce.
- Affected commit/version and environment details.

We will acknowledge receipt, triage severity, and provide a remediation timeline when possible.
